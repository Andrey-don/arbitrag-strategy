⚙️Настройка — одноразово

1
Получи токен T-Invest (Read-only) Зайди на tinkoff.ru/invest → Настройки → API → Создать токен.
Торговые права не нужны — выбирай Read-only.
2
Открой файл .env в папке проекта 
TINKOFF_TOKEN=t.твой_токен_здесь
3
Сохрани файл — больше этот шаг не повторяется.
✅ Токен нужен только для загрузки данных. После того как CSV-файлы скачаны — интернет и токен больше не нужны. Бэктестер работает полностью офлайн.

📥Загрузка истории

✅ Самый простой способ: Двойной клик на download_history.bat на рабочем столе или в папке проекта.

Скрипт выполнит два шага:

1
Загрузит 1-мин и 5-мин с T-Invest API ~90 запросов на таймфрейм, 2–3 минуты. Показывает прогресс и размер файлов.
2
Сгенерирует 15-мин и 1-час из 5-мин Пересемплирование, занимает секунду. Файлы появятся в data\history\.
⚠️ Обновление истории Запускай загрузчик повторно — он перезаписывает файлы. Актуальная история всегда будет за последние 3 месяца от сегодняшней даты.

🔍Вкладка «🔍 Оптимизатор»

Автоматически перебирает все комбинации параметров и находит лучшую.

Процесс оптимизации

1
Задай диапазоны параметров entry_scalp (мин/макс/шаг), stop_add, значения target.
2
Выбери метрику оптимизации Sharpe — стабильность, ROI — абсолютная доходность, Winrate — процент побед.
3
Нажми «🔍 Запустить оптимизацию» Прогресс-бар. Топ-10 комбинаций + тепловые карты.
💡 Стратегия оптимизации в 2 шага:
1. Широкий поиск с шагом 1.0 — находим область.
2. Точный поиск с шагом 0.5 вокруг лучшей точки — уточняем параметры.
⚠️ Опасность переобучения (overfitting). Найденные параметры работают хорошо на обучающем периоде. Обязательно проверяй на другом периоде (out-of-sample).

📡Сигналы стратегии ТАТки

СигналУсловиеДействие
LONG_GOOD Спред ≥ 7 ₽ И Ratio ≥ 1.011 Сильный вход — приоритет
LONG_SCALP Спред ≥ 5 ₽ И Ratio ≥ 1.008 Скальп-вход — при R/R ≥ 2
ALERT Спред ≥ 2.5 ₽ Наблюдать, готовить заявки
EXIT Спред ≤ 0.5 ₽ Линии сошлись — закрывать!
TAKE_PROFIT Спред ≥ 20 ₽ Зона выхода на среднесрочке
STRONG_LONG Ratio < 1.0 (прив. дороже обычки) Аномалия — сильный вход
ℹ️ Фильтр R/R ≥ 2.0
Алгоритм не входит, если расчётное соотношение риск/прибыль меньше 2:1, даже при наличии сигнала. Это ключевой фильтр для отсева плохих входов.

💡Советы по работе

1. Workflow: Оптимизатор → проверка → Тренажёр Сначала найди параметры через Оптимизатор на 2 месяцах → проверь на третьем месяце → потренируйся в Тренажёре с этими параметрами.
2. Вечерняя сессия прибыльнее Если анализ сессий показывает преимущество вечера — фильтруй данные (Фильтр от 19:00 до 23:59) и тестируй отдельно.
3. Слипадж 0.2–0.5 ₽ даёт реалистичную картину Рыночные ордера на ТАТках в стакане двигают цену на 0.1–0.3 ₽. Без слиппажа результаты будут завышены.
4. Не торгуй реальными деньгами только по бэктесту Бэктест не учитывает: ликвидность стакана, проскальзывание при крупных объёмах, задержки исполнения, новостные гэпы. Тренажёр ближе к реальности.

❓Частые вопросы

Сколько хранится история в CSV-файлах?

По умолчанию 3 месяца от сегодняшней даты. Чтобы изменить — открой src\monitoring\download_history.py и измени:

MONTHS_BACK = 3  # измени на 6 для полугода

Учти: 6 месяцев 1-мин данных = ~180 запросов к API, ~5–7 минут загрузки.

Почему мало данных при 1-мин таймфрейме?

T-Invest API отдаёт максимум 1 день за запрос на минутных таймфреймах. Загрузчик делает ~90 запросов за 3 месяца — это нормально, просто занимает время. Следи за прогрессом в консоли.

Можно ли добавить данные к существующим (не перезаписывать)?

Пока нет — загрузчик перезаписывает файлы полностью. Просто запускай download_history.bat раз в месяц — получишь актуальную историю за последние 3 месяца.

Бэктестер показывает «Нет данных» при загрузке из CSV

Проверь:

1. Файлы существуют в data\history\ — запусти download_history.bat.

2. В бэктестере выбран источник «💾 Локальный CSV».

3. Фильтр дат не выходит за пределы данных в файле.

Как тестировать другую пару (SBER/SBERP, GAZP)?

1. Добавь тикеры в download_history.py → INSTRUMENTS.

2. Запусти загрузчик — файлы появятся в data\history\.

3. В backtest.py потребуется небольшая доработка: изменить пороги сигналов (спред, ratio) под характеристики новой пары.

Бэктестер и дашборд конфликтуют между собой?

Нет — они работают на разных портах: бэктестер на :8502, дашборд на :8501. Можно запускать оба одновременно.
