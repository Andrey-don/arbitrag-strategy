### Урок 01 — Макроэкономика для криптотрейдеров

**Что это:** Связь биткоина с фондовым рынком США и макроэкономическими циклами. Институционализация крипторынка, режимы Risk-on/Risk-off, ключевые индикаторы.

**Ключевые цифры из урока:**
- Корреляция биткоина с Рассел 2000: **84%** (пик **93%**)
- Корреляция биткоина с Nasdaq: **72%** (пик **86%**)
- Корреляция биткоина с медью: **70%** (пик **94%**)
- Институционалы: **25%** всех сделок, ритейл: **8%**
- Фондовый рынок **80% времени** торгуется на опережающих макрофакторах

**Плюсы:**
- Даёт системное понимание «почему крипта растёт/падает» — без этого слепо торговать
- Конкретные индикаторы для отслеживания: ставка ФРС, М2, кривая доходности, кредитный спред, IPO-календарь
- Честно объясняет: биткоин — это не «цифровое золото», а High Beta Small Cap циклический актив
- Важный практический вывод: в выходные Уолл-стрит волатильность биткоина затухает

**Минусы / ограничения:**
- Урок про среднесрочную торговлю, не про арбитраж — применимость к боту косвенная
- Все «что делать боту» остались без ответа — нет конкретики для автоматизации
- Нет кода, нет API, нет конкретных источников данных для программного получения индикаторов

**Применимость в боте:** Низкая (прямая) / Высокая (контекстная)

**Вывод:** Обязательный фундамент для понимания рыночного контекста. Даёт понять КОГДА арбитражный бот будет работать хорошо (Risk-on, ликвидность высокая, волатильность умеренная) и КОГДА нужно снижать активность (Risk-off, выходные, перед важными макрособытиями). Прямого кода для бота нет — это урок «картины мира».

### Урок 02 — Научил Новичка ТРЕЙДИНГУ за День. Руководство 1

**Что это:** Пошаговая практика сбора спреда на неликвидных токенах биржи третьего эшелона. Стакан, маркетмейкер, лимитные ордера, API-настройка.

**Ключевые параметры из урока:**
- Объём одной сделки: **~$2** (мелко, чтобы не пугать маркетмейкера)
- Депозит на старт: пополнение спотового кошелька **USDT в сети TRC20**
- Биржи: **MEXC, Gate, KuCoin, HTX, Bybit, Bitget** (3-й эшелон)
- API-права: только чтение + ордера, **трансферы запрещены**
- **99%** неудач — на этапе настройки платформы и API

**Плюсы:**
- Первый урок с реальным кодом — скелет `ccxt.async_support` уже применим
- Конкретные биржи и инструменты — сразу понятно что подключать
- Механика «мейкер/тейкер» и работа со стаканом — фундамент для арбитражного бота
- Прямо указывает: этот навык — основа для перехода к **межбиржевому арбитражу**

**Минусы / риски:**
- Фокус на ручном трейдинге в терминале — код нужно строить самостоятельно
- Неликвидные «скам-токены» — высокий риск застрять в позиции без контрагента
- Остатки позиции копятся — бот обязан отслеживать точный баланс после каждой сделки
- Маркетмейкер может уйти в любой момент — необходима логика остановки

**Что добавить в бота:**
- `fetch_order_book()` для получения bid/ask в реальном времени
- `create_limit_buy_order()` / `create_limit_sell_order()` по границам спреда
- `cancel_all_orders()` как аналог кнопки «Пробел»
- Проверка баланса базового актива перед каждым sell-ордером

**Вывод:** Самый прикладной урок из трёх. Это прямая механика будущего бота — купить внизу спреда, продать вверху. Трейдер сам говорит: «это первый шаг, потом будет несколько бирж» — значит следующие уроки выведут на межбиржевой арбитраж.

### Урок 03 — Научил Новичка ТРЕЙДИНГУ за День. Руководство 2.0

**Что это:** Межбиржевой фьючерсный арбитраж. Две ноги (Long + Short) на разных биржах, ожидание схождения ratio, управление фандингом и плечом.

**Ключевые параметры из урока:**
- Порог входа: **3–3.5%** расхождения ratio
- Максимальное плечо: **×3–×4** (не выше!)
- Размер одной ноги: **~$100**
- Фандинг: **0.3%** за отсечку — выходить до таймера при прибыли **1%**
- Схождение происходит в **99% случаев** (если токен не делистнут)
- Усреднение при расширении спреда до **5%** — норма и преимущество арбитража

**Плюсы:**
- Полная механика работающей стратегии — от сигнала до выхода
- Конкретный ccxt-скелет с `asyncio.gather` для двух бирж одновременно
- Чёткие параметры входа, плеча, фандинга — можно напрямую кодировать в бота
- Указан следующий уровень: MOEX-арбитраж и «тройнички» (спот + фьючерс)

**Минусы / риски:**
- Ликвидация при расширении спреда + высокое плечо — критический риск
- Фандинг в момент отсечки может резко изменить P&L
- REST API слишком медленный для продакшна — нужен WebSocket (`ccxt.pro`)
- Зависимость от скринера — без него сложно найти возможности вручную

**Что добавить в бота:**
- `fetch_funding_rate()` — проверять перед каждым входом
- `fetch_positions()` — отслеживать liquidation price в реальном времени
- Цикл мониторинга после открытия позиции: `ratio <= 1.01` → закрыть обе ноги
- Заменить `asyncio.sleep(1)` на WebSocket через `ccxt.pro`

**Вывод:** Ключевой урок всего курса — здесь собрана полная стратегия межбиржевого арбитража. Все три урока дают законченную картину: макроконтекст (урок 01) → механика стакана (урок 02) → **полноценный арбитраж** (урок 03). Можно приступать к разработке бота.

### Урок 04 — Топ 7 Приложений Трейдера

**Что это:** Обзор рабочего фреймворка трейдера: TradingView, Привод Бондаря, MT5, CScalp, Remote Desktop, Telegram, Trust Wallet. Какие заменяет бот, какие остаются ручными.

**Ключевые параметры из урока:**
- TradingView: **$150–180** по скидкам (ЧП, Рамадан), **$480** без скидки
- CScalp/Metascalp: коннектится к **~15 биржам** по API
- Трейдинг — **«$1 × 1 000 000 раз»**, не «1000 иксов»
- Дневник трейдера в арбитраже: **не нужен** — статистика безупречная

**Плюсы:**
- Даёт полную картину экосистемы — понятно что нужно строить, а что уже готово
- Чёткое разделение: бот заменяет CScalp и Telegram-скринер полностью
- Telegram API = канал уведомлений бота — приоритет при разработке

**Минусы / ограничения:**
- Урок обзорный — нет новой механики торговли, только инструменты
- Стоимость и доступность большинства инструментов не уточнена

**Что добавить в бота:**
- `python-telegram-bot` или `aiogram` → уведомления о сигналах и сделках
- `fetch_balance()` → автоматический мониторинг балансов на всех биржах
- Бот = CScalp + скринер + Telegram в одном процессе

**Вывод:** Полезный обзорный урок. Главный инсайт для разработки: **бот должен заменить CScalp и Telegram-скринер** — это два ключевых инструмента, которые сейчас делаются вручную. Всё остальное (TradingView, переводы) остаётся ручным.

### Урок 05 — Худшие стратегии ТРЕЙДИНГА, на которых теряют деньги

**Что это:** Антипаттерны — 8 провальных стратегий из интернета с разбором механики убытка. Подтверждение правильности арбитражного подхода: эксплуатация неэффективности, а не угадывание направления.

**Ключевые параметры из урока:**
- Рабочий диапазон рубля: **50–120** (единственный рабочий Range Trading-кейс)
- Мартингейл: работает ТОЛЬКО как вспомогательный инструмент внутри рабочей стратегии
- Красный флаг №1: слово **«замок»** — немедленно уходить
- Боты на индикаторах: из **999 триллионов** созданных ни один не работает стабильно

**Плюсы:**
- Подтверждает правильность выбранной стратегии — арбитраж эксплуатирует реальную неэффективность
- Чёткий критерий проверки стратегии: есть ли конкретная неэффективность и алгоритм?
- Даёт конкретные красные флаги для распознавания мусорных стратегий

**Минусы / ограничения:**
- Урок обзорный — нет новой механики торговли
- Типичные убытки по стратегиям не уточнены (помечены [НЕЯСНО])

**Что добавить в бота:**
- Никаких индикаторов (MA, RSI и т.д.) как фильтров входа
- Никакого мартингейла при убыточной ноге — только стоп
- Никакого «замка» на одной бирже — только разные биржи
- Защитная проверка: вход только при математически доказанном спреде

**Вывод:** Урок-валидация. Подтверждает, что арбитражный бот строится правильно. Главный инсайт: бот должен эксплуатировать конкретную рыночную неэффективность (расхождение цен), игнорируя всё остальное. Индикаторы, тренды, Smart Money — в корзину.

### Урок 06 — Научил Новичка ТРЕЙДИНГУ за День. Подробное руководство

**Что это:** Практический туториал от нуля: регистрация на MEXC, настройка CScalp, мышка/клавиатура, алгоритм скальпинга маркетмейкера. Результат новичка — **~100 сделок**, **$0.40** прибыли при объёме **$2**, ни одного убытка.

**Ключевые параметры из урока:**
- Рабочий объём: **$1–2** (маркетмейкер не заметит)
- Старт депозита: **$50** USDT в сети **TRC20**
- Биржа: **MEXC** (лояльна без санкций)
- Терминал: **CScalp / Tiger Trade / Metascalp**
- Левая кнопка мыши → **buy**, Правая → **sell**
- **Пробел** → отмена всех ордеров, **Alt** → захват всего остатка

**Плюсы:**
- Самый детальный операционный урок курса — каждое физическое действие расписано
- Конкретный алгоритм выбора токена (широкий спред + только маркетмейкер в стакане)
- Код `scalp_market_maker()` — готовый скелет с исправленным багом `bids[0][0]`
- Объясняет ПОЧЕМУ малый объём критичен — математика маркетмейкера

**Минусы / ограничения:**
- 2 [НЕЯСНО] требуют скриншотов: расположение панелей терминала + настройки мыши
- DPI мышки не упомянуто
- Условия досрочного выхода в минус не раскрыты

**Что добавить в бота:**
- `fetch_order_book()` → выбор токена с широким спредом и только одним участником
- `create_limit_buy_order()` / `create_limit_sell_order()` по краям спреда
- `fetch_balance()[token]['free']` вместо фиксированного amount (аналог Alt)
- `cancel_all_orders()` при зависшем ордере (аналог Пробела)

**Вывод:** Операционное руководство — из него напрямую вырастает код `src/strategies/` и `src/execution/`. Два оставшихся [НЕЯСНО] закрываются скриншотами в Google Doc → NotebookLM (бесплатно, без расхода токенов Claude).

## Матрица рисков арбитража

_Заполняется по мере изучения уроков. Сейчас — базовые риски любого арбитража._

| Риск | Вероятность | Влияние | Как митигировать |
|------|------------|---------|-----------------|
| Спред исчезает до исполнения | Высокая | Критично | Скорость: websocket + asyncio |
| Комиссии съедают прибыль | Высокая | Высокое | Порог входа > комиссии × 2 |
| Частичное исполнение ордера | Средняя | Высокое | Проверять filled qty |
| Сбой API одной биржи | Средняя | Критично | Retry с backoff, хеджирование |
| Разный объём на биржах | Средняя | Высокое | Проверять ликвидность до входа |
| Funding rate (крипта) | Высокая | Среднее | Учитывать в расчёте спреда |

### Урок 11 — Арбитраж на МОЕХ.Часть 1: Введение в ТАТки

**Что это:** Первое введение в MOEX-арбитраж. Пара ТАТНн/ТАТНп (Татнефть обычка + прив). Построение графика спреда в TradingView. Механика стратегии.

**Ключевые параметры:**
- Инструмент: TATN / TATNP (МОЕХ, ММВБ)
- Типичный спред: 2–5 рублей (скальп) / до 20 руб (среднесрочно)
- Начальный капитал: 100 тыс. ₽
- Цель новичка: ×10–15 за 1.5 месяца
- Брокер: CScalp (Коскэп), Привод
- НДФЛ: полное сальдирование лонг+шорт

**Плюсы:**
- Доступна из РФ без ограничений — полностью рублёвая
- Низкий порог входа, подтверждена на десятках учеников
- Коинтегрированная пара — статистическое преимущество
- Нет зависимости от крипто-инфраструктуры

**Риски:**
- Структурный сдвиг (уравнивание дивидендов уже произошло, что если снова изменится?)
- Маржин-колл при неправильном плече

**Применимость в боте:** Высокая — MOEX ISS API бесплатен, данные доступны

**Вывод:** Стратегия №1 для жителя РФ. Подтверждена практикой. Идеальная точка старта для разработки бота.

### Урок 12 — Арбитраж на МОЕХ.Часть 2: Практика в стакане

**Что это:** Живая практика в Приводе. Настройка окон, стаканов, реальные входы и выходы.

**Ключевые параметры:**
- «Прокидки» прива: 1 000–2 000 лотов за раз, до 1 рубля движение
- Для обычки нужно 5 000–10 000 лотов
- Выставлять лимитки за 0.5–1 рубль от цены
- Escape = отменить все заявки (настроить горячую клавишу)

**Плюсы:**
- Прокидки создают предсказуемые точки входа/выхода
- Налоговая эффективность: лонг+шорт = сальдо

**Риски:**
- Нужен опыт чтения стакана
- Одну ногу закрыл — вторая может уйти против

**Применимость в боте:** Средняя (логика прокидок сложно формализуется)

**Вывод:** Тактика выхода через прокидки — освоить руками, затем попробовать автоматизировать.

### Урок 13 — Теория спреда и ТАТки: Два режима торговли

**Что это:** Теоретическая база статистического арбитража. История пары. Два режима: скальп и среднесрочка.

**Ключевые параметры:**
- История: до 2019 — 250 рублей спред; после 2020 — 5–20 рублей
- Пол спреда исторически: ~0 рублей (ниже не ходит)
- «Ноль» у ТАТок = 5–6 рублей (там они чаще всего)
- Усреднение в арбитраже = инструмент (не мартингейл!)
- Скальпер: реагирует на скорость, не на уровень
- Среднесрочник: держит неделями, ждёт 5–20 рублей

**Плюсы:**
- Чёткое разграничение двух торговых стилей
- Усреднение обосновано (коинтеграция)

**Риски:**
- «99 прибыльных сделок, одна отрицательная может порвать»
- Шорт спреда → потенциально неограниченный убыток

**Применимость в боте:** Высокая — оба режима можно закодить

**Вывод:** Обязательная теоретическая база. Определить какой режим автоматизировать первым.

### Урок 14 — Набор позиции и выход (Часть 2)

**Что это:** Практика работы с крупными объёмами. Алерты. Стратегии выхода.

**Ключевые параметры:**
- Алерт на 2.5 рубля в TradingView (пересечение)
- Комиссии: 4× на круг (вход+выход × 2 ноги)
- Набирать частями по 250–1 000 лотов
- Заявки выхода: расставить заранее, не ждать движения
- Прив: «прокидывают чаще и глубже»

**Применимость в боте:** Высокая (алерт + автовыставление заявок)

**Вывод:** Алерт-система — первая функция для автоматизации.

### Урок 15 — Комментарий по ТАТкам: Риски и Маржин-колл

**Что это:** Короткий урок о реальном маржин-колле (40% депо) и правильном управлении рисками.

**Ключевые параметры:**
- Макс сделок: 3 в день (у автора)
- Только на «хороших движениях», не на волатильности
- При приближении маржи: закрывать вручную!
- Автозакрытие = сквиз = большая потеря

**Плюсы:**
- Честный разбор реальной ошибки

**Риски:**
- Агрессивное плечо → маржин-колл
- Шорт прива опаснее лонга (исторически)

## Главные выводы

### Что НЕ делать (красные флаги)

- Запускать бот на реальные деньги без DRY_RUN минимум 1–2 недели
- Игнорировать маржу — при приближении закрывать ВРУЧНУЮ
- Брать шорт спреда агрессивно без учёта истории (спред мог быть 250 рублей)
- «Бить в свою же заявку» на разных пропсчетах одного брокера (ЦБ)
- Выходить по рынку (taker) там, где можно лимиткой (maker)

### Что стоит внедрять (приоритеты)

1. **MOEX ISS API** — получать котировки TATN/TATNP в реальном времени
2. **Алерт-бот** — Telegram-уведомление при спреде < порога (скальп) или > порога (выход)
3. **Расчёт спреда** → ratio (A/B) и difference (A−B)
4. **Мониторинг маржи** — предупреждение при приближении к margin call
5. **Самописный скринер пар** — подобрать другие коинтегрированные пары MOEX
