Урок 02 Научил Новичка ТРЕЙДИНГУ за День. Руководство 1

Стакан, спред, маркетмейкер на неликвидных токенах. Биржи 3-го эшелона, API-ключи, первый ccxt-скелет.

📊 Ключевые параметры
  • Объём сделки: ~$2 — мелко, чтобы не пугать маркетмейкера
  • Биржи: MEXC, Gate, KuCoin, HTX, Bybit, Bitget (3-й эшелон)
  • Депозит: USDT в сети TRC20
  • API: только чтение + ордера, трансферы — запрещены
  • 99% неудач — на этапе настройки платформы и API-ключей
✅ Плюсы
  • Первый урок с реальным кодом — скелет ccxt.async_support уже применим
  • Конкретные биржи — сразу понятно что подключать через ccxt
  • Механика мейкер/тейкер и стакан — фундамент арбитражного бота
  • Прямо говорит: этот навык — основа для перехода к межбиржевому арбитражу
⚠️ Риски
  • Неликвидные «скам-токены» — риск застрять в позиции без контрагента
  • Остатки позиции копятся — бот обязан отслеживать точный баланс после каждой сделки
  • Маркетмейкер может уйти в любой момент — нужна логика остановки
📌 Что добавить в бота
  • fetch_order_book() — получить bid/ask в реальном времени
  • create_limit_buy_order() / create_limit_sell_order() по границам спреда
  • cancel_all_orders(symbol) — аналог кнопки «Пробел»
  • Проверка баланса базового актива перед каждым sell-ордером
Вывод: Самый прикладной урок — прямая механика будущего бота. Трейдер сам говорит: «это первый шаг, потом будет несколько бирж» — следующие уроки выведут на межбиржевой арбитраж.
Применимость: Высокая
Урок 03 Научил Новичка ТРЕЙДИНГУ за День. Руководство 2.0

Межбиржевой фьючерсный арбитраж. Ratio, две ноги (Long+Short), фандинг, плечо ×3–4, asyncio.gather.

📊 Ключевые параметры
  • Порог входа: 3–3.5% расхождения ratio
  • Максимальное плечо: ×3–×4 (не выше!)
  • Размер одной ноги: ~$100
  • Фандинг: 0.3% за отсечку → выходить при прибыли 1% до таймера
  • Схождение в 99% случаев (если токен не делистнут)
✅ Плюсы
  • Полная механика стратегии от сигнала до выхода
  • Готовый ccxt-скелет с asyncio.gather для двух бирж одновременно
  • Чёткие параметры входа, плеча, фандинга — напрямую в код
  • Усреднение при расширении до 5% — норма и преимущество арбитража
⚠️ Риски
  • Ликвидация при расширении спреда + высоком плече — критично
  • Фандинг в момент отсечки может резко изменить P&L
  • REST API слишком медленный для продакшна — нужен WebSocket (ccxt.pro)
  • Без скринера сложно найти возможности — нужна инфраструктура мониторинга
📌 Что добавить в бота
  • fetch_funding_rate() — проверять перед каждым входом
  • fetch_positions() — отслеживать liquidation price в реальном времени
  • Цикл мониторинга: ratio <= 1.01 → закрыть обе ноги
  • Заменить asyncio.sleep(1) на WebSocket через ccxt.pro
Вывод: Ключевой урок всего курса — полная стратегия межбиржевого арбитража. Три урока дают законченную картину: макроконтекст → механика стакана → полноценный арбитраж. Можно приступать к разработке бота.
Применимость: Очень высокая

Урок 01 Макроэкономика для криптотрейдеров

Risk-on/Risk-off, ставка ФРС, М2, корреляция биткоина с Рассел 2000. Почему крипта зависит от Уолл-стрит.

📊 Ключевые цифры
  • Биткоин ↔ Рассел 2000: 84% (пик 93%) — лучшая корреляция
  • Биткоин ↔ Nasdaq: 72% (пик 86%)
  • Биткоин ↔ Медь: 70% (пик 94%) — ранний индикатор цикла
  • Институционалы: 25% сделок, ритейл: 8%
  • Рынок торгуется на макрофакторах в 80% времени
✅ Плюсы
  • Системное понимание «почему крипта растёт/падает» — без этого слепо торговать
  • Конкретные индикаторы: ставка ФРС, М2, кривая доходности, кредитный спред, IPO-календарь
  • Честный вывод: биткоин — не «цифровое золото», а High Beta Small Cap циклический актив
  • В выходные Уолл-стрит волатильность биткоина затухает → снижать активность бота
❌ Ограничения
  • Урок про среднесрочную торговлю, не про арбитраж — прямая применимость к боту косвенная
  • «Что делать боту» по каждому индикатору — без ответа, нет конкретики для автоматизации
  • Нет кода, нет API, нет источников данных для программного получения индикаторов
📌 Применение в боте
  • Risk-on + высокая ликвидность М2 → бот активен, пороги можно снизить
  • Risk-off / перед заседанием ФРС → остановить бот или поднять минимальный спред
  • Выходные Уолл-стрит → снизить активность, волатильность низкая
Вывод: Обязательный фундамент — даёт понять КОГДА бот будет работать хорошо и КОГДА снижать активность. Прямого кода нет — это урок «картины мира».
Применимость: Косвенная (контекст) / Прямая в коде: отсутствует

Красные флаги — что НЕ делать

🚨 Запускать бот на реальные деньги без проверки в DRY_RUN минимум 1–2 недели
🚨 Игнорировать скорость исполнения — в арбитраже миллисекунды решают всё
🚨 Не учитывать комиссии при расчёте минимального порога спреда
🚨 Оставлять бот без watchdog и автоматического стопа по дневному убытку
🚨 Не обрабатывать частичное исполнение ордера — может создать неприкрытую позицию
