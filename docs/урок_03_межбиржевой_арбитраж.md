# Урок 03: Научил Новичка ТРЕЙДИНГУ за День. Подробное руководство 2.0

## 1. Краткое резюме

Переход от сбора спреда на одной бирже (урок 02) к полноценному межбиржевому арбитражу. Трейдер учит зарабатывать на расхождении цен одного и того же актива между двумя разными биржами через фьючерсные контракты. Вместо физического перевода монет — одновременное открытие разнонаправленных позиций (Long и Short) и ожидание схождения цен. Схождение происходит в **99% случаев**.

---

## 2. Ключевые концепции

- **Межбиржевой фьючерсный арбитраж** — покупка актива там, где дешевле, и одновременная продажа там, где дороже, без физического перевода монет.
- **Использование фьючерсов** — позволяет открывать шорт и использовать плечо. Для операции на **$1000** достаточно иметь **$100** обеспечения (плечо **×10**).
- **График спреда (Ratio)** — отношение цены на бирже A к цене на бирже B. В эффективном рынке = 1. Отклонение на **3%** (0.97 или 1.03) — это арбитражная возможность.
- **Ноги арбитража** — два одновременных действия: лонг (покупка) на «дешёвой» бирже + шорт (продажа) на «дорогой».
- **Фандинг (Ставка финансирования)** — механизм биржи удерживать фьючерс близко к споту. При долгом удержании позиции съедает прибыль (например, **0.3%** от суммы позиции за каждую отсечку).
- **Схождение (Convergence)** — в **99% случаев** цены на разных биржах выравниваются. Задача: дождаться схождения или забрать частичный возврат (например, **1%**) и закрыть обе ноги.

---

## 3. Алгоритм стратегии

1. **Получить сигнал** — скринер уведомляет о расхождении цен на тикер (например, MEXC и Bybit разошлись на **3.5%**).
2. **Оценить график спреда** — в TradingView построить ratio двух цен. Оценить историческую волатильность спреда: если спред часто достигал **7%**, заходить при **2%** может быть рано.
3. **Проверить фандинг** — посмотреть ставку и таймер до отсечки на обеих биржах. Убедиться, что фандинг не съест прибыль.
4. **Подготовить стаканы** — открыть два стакана в терминале. Выбрать одинаковый объём (например, по **$100** на каждую ногу).
5. **Войти одновременно** — левая кнопка (купить лонг) на дешёвой бирже + правая (шорт) на дорогой.
6. **Сопровождать** — ждать схождения ratio → 1. Если позиция даёт **1% за несколько минут** — закрывать, не дожидаясь полного схождения.
7. **Выйти** — Alt (закрыть всю позу по рынку) + клик в оба стакана одновременно. Зафиксировать прибыль.

## 5. Шпаргалка: термин → объяснение → применение в боте

| Термин | Что означает | Как реализовать в Python |
|--------|-------------|------------------------|
| График спреда (Ratio) | Отношение цены A к цене B | `ratio = price_A / price_B` → если `ratio > 1 + threshold`: сигнал |
| Фандинг | Периодическая комиссия держателям позиций | `exchange.fetch_funding_rate(symbol)` → не входить если отсечка близко и ставка невыгодна |
| Шорт | Продажа контракта без наличия актива | `create_order(..., 'sell', params={'positionSide': 'SHORT'})` |
| Ликвидация | Принудительное закрытие при исчерпании обеспечения | `fetch_positions()` → отслеживать liquidation price, ставить stop-loss |
| Закрытие ноги | Мгновенное закрытие позиции по рынку | Получить `position['contracts']` → отправить обратный рыночный ордер |

## 6. Важные замечания

- **Риск ликвидации:** с плечом **×10** при расширении спреда на **10%** шортовую ногу может ликвидировать. Рекомендуемое плечо: не выше **×3–×4** (на **$1000** позиция не более **$4000**).
- **Риск фандинга:** в момент отсечки рынок может резко дёрнуться. Лучше забирать **1%** и выходить до таймера, чем держать и рисковать.
- **Усреднение — это норма:** если спред расширился с **3%** до **5%** — можно добрать позицию. Математическое схождение к 1 неизбежно (если токен не делистнут).
- **Отличие от урока 02:** не нужно обманывать маркетмейкера в одном стакане. Используются рыночные или агрессивные лимитные удары в стаканы **двух бирж одновременно**.

## 8. Применение в арбитражном боте

**Биржи:** мультибиржевое подключение через `ccxt` с `defaultType: swap` (фьючерсы).

**Синхронизация стаканов:** `asyncio.gather()` для параллельного запроса с двух бирж.

**Обнаружение возможности:** `bid_A / ask_B > 1 + threshold` → вход.

**Исполнение:** одновременно `create_market_buy_order` на дешёвой + `create_market_sell_order` на дорогой через `asyncio.gather`.

```python
import ccxt.async_support as ccxt
import asyncio

THRESHOLD = 0.03   # минимальное расхождение для входа — 3%
MAX_LEVERAGE = 3   # не выше ×3 по рекомендации трейдера
TRADE_SIZE = 100   # объём каждой ноги в USDT

async def monitor_and_arbitrage():
    mexc  = ccxt.mexc( {'apiKey': 'KEY', 'secret': 'SEC', 'options': {'defaultType': 'swap'}})
    bybit = ccxt.bybit({'apiKey': 'KEY', 'secret': 'SEC', 'options': {'defaultType': 'swap'}})

    symbol = 'BTC/USDT:USDT'

    try:
        while True:
            # 1. Параллельный запрос стаканов с двух бирж
            ob_mexc, ob_bybit = await asyncio.gather(
                mexc.fetch_order_book(symbol, limit=5),
                bybit.fetch_order_book(symbol, limit=5),
            )

            mexc_ask  = ob_mexc['asks'][0][0]   # лучшая цена покупки на MEXC
            bybit_bid = ob_bybit['bids'][0][0]  # лучшая цена продажи на Bybit

            # 2. Расчёт ratio: если Bybit дороже MEXC — покупать на MEXC, шортить на Bybit
            ratio = bybit_bid / mexc_ask

            if ratio >= 1 + THRESHOLD:
                print(f"Арбитраж найден! Ratio={ratio:.4f}")

                # 3. Проверить фандинг перед входом
                fr_mexc, fr_bybit = await asyncio.gather(
                    mexc.fetch_funding_rate(symbol),
                    bybit.fetch_funding_rate(symbol),
                )
                # Если фандинг невыгоден — пропустить

                # 4. Одновременный вход в обе ноги
                await asyncio.gather(
                    mexc.create_market_buy_order(symbol, TRADE_SIZE),   # лонг на MEXC
                    bybit.create_market_sell_order(symbol, TRADE_SIZE), # шорт на Bybit
                )
                print("Позиция открыта. Ожидаем схождения...")

                # 5. Мониторинг и выход при схождении (ratio <= 1.01 или прибыль >= 1%)
                # TODO: реализовать цикл мониторинга позиции
                break

            await asyncio.sleep(1)  # пауза для REST API (в продакшне — websocket)

    except Exception as e:
        print(f"Ошибка: {e}")
    finally:
        await mexc.close()
        await bybit.close()

if __name__ == '__main__':
    asyncio.run(monitor_and_arbitrage())
```

> **Важно:** для продакшна REST `asyncio.sleep(1)` заменить на **WebSocket** через `ccxt.pro` — это единственный способ получать данные с минимальной задержкой.
