# Урок 10: Как создать синтетические арбитражные пары в TradingView

> **Источник документа:** `E:\Антон Клевцов Трейдинг\Курсы\[Антон Клевцов] Грамотный трейдинг (2024)\3. Арбитраж\7. Инструкция по созданию синтетических арбитражных криптовалютных пар_v2\7. Инструкция...txt`
> Конспект пронумерован **10**, потому что это десятый обработанный урок по порядку. Сам документ в курсе носит номер **7** (раздел 3.7). Не путать.

> **Суть:** собрать все спотовые тикеры Binance и Bybit, найти пересечение, сформировать список синтетических пар в формате `BINANCE:TICKER/BYBIT:TICKER` и импортировать в TradingView для мониторинга спредов.
> **Инструменты:** TradingView screener + watchlist, Excel, Блокнот.
> **Время:** ~15 минут на весь процесс.

---

## 1. Зачем нужны синтетические пары?

Синтетическая пара вида `BINANCE:BTCUSDT/BYBIT:BTCUSDT` в TradingView показывает **отношение цен** одного актива на двух биржах. График такой пары:
- При спреде 0% — линия равна 1.0
- При росте спреда — отклонение от 1.0 вверх или вниз
- Это быстрый визуальный скринер арбитражных возможностей без специализированного ПО

Используется как первичный фильтр перед входом в позицию.

---

## 2. Шаг 1 — Собрать тикеры Binance (спот, USDT)

1. Открыть **TradingView → Screener** (скринер)
2. Выбрать биржу **Binance**
3. Применить фильтр: **Spot**, пара с **USDT**
4. Список найдёт ~386 тикеров

> ⚠️ **Критический нюанс:** TradingView по умолчанию подгружает только первые 150 результатов. Нужно **прокрутить список до самого конца** — дождаться загрузки всех 386 тикеров.

5. Выбрать любой тикер из списка → нажать **Ctrl+A** (выделить все)
6. Нажать **«Добавить выбранные символы в список»** → сохранить как **«Binance»**

---

## 3. Шаг 2 — Собрать тикеры Bybit (спот)

1. В скринере переключить биржу на **Bybit**
2. Фильтр: Spot
3. Список: ~465 тикеров
4. Прокрутить до конца (то же правило — иначе загрузятся только первые 150)
5. Ctrl+A → **«Добавить в список»** → сохранить как **«Bybit»**

---

## 4. Шаг 3 — Экспортировать списки в файлы

1. Перейти в **расширенный режим** вотчлиста
2. Экспортировать список **Binance** → сохранить как `binance.txt`
3. Экспортировать список **Bybit** → сохранить как `bybit.txt`

**Формат файла** (пример содержимого):
```
BINANCE:BTCUSDT, BINANCE:ETHUSDT, BINANCE:SOLUSDT, ...
```
Записи идут через запятую, без пробелов.

---

## 5. Шаг 4 — Обработка в Excel

### 5.1 Загрузить оба файла

1. Excel → **Данные → Получить внешние данные → Из текста**
2. Открыть `binance.txt`
3. Тип данных: **с разделителем** → разделитель: **запятая**
4. Формат: **общий** → вставить с **начала листа** → OK
5. Повторить для `bybit.txt` → вставить в **строчку 3** (или в следующую строку)

Результат: тикеры вставятся **по горизонтали** (в строку).

### 5.2 Транспонировать строки в столбцы

1. Выделить все тикеры Binance (Shift + Ctrl+End для выделения диапазона)
2. Ctrl+C (копировать)
3. Выбрать пустую ячейку (например, A10)
4. **Главная → Вставить → Транспонировать**

Повторить для Bybit → вставить в соседний столбец (B).

Теперь у нас два вертикальных столбца: **Столбец A** = тикеры Binance, **Столбец B** = тикеры Bybit.

### 5.3 Удалить префиксы бирж

Нам нужны только названия инструментов без приставки биржи:

1. **Ctrl+H** (Найти и заменить)
2. Найти: `BYBIT:` → Заменить на: *(пусто)* → **Заменить всё** (465 замен)
3. Найти: `BINANCE:` → Заменить на: *(пусто)* → **Заменить всё**

В столбцах останутся только чистые имена: `BTCUSDT`, `ETHUSDT` и т.д.

---

## 6. Шаг 5 — Найти общие тикеры (пересечение списков)

Используем функцию **СЧЁТЕСЛИ** для поиска совпадений.

**В столбце C** (рядом с Binance):
```excel
=СЧЁТЕСЛИ($B:$B; A1)
```
- Если тикер из столбца A есть в столбце B → вернёт `1`
- Если нет → вернёт `0`

Двойной клик по правому нижнему углу ячейки — протянуть формулу до конца списка.

**В столбце D** (рядом с Bybit):
```excel
=СЧЁТЕСЛИ($A:$A; B1)
```
Протянуть аналогично.

**Применить фильтр:**
- Выделить строку заголовков → Данные → Фильтр
- В столбце C: показать только `1`
- В столбце D: показать только `1`

Остались только тикеры, которые **есть на обеих биржах** (~400 пар).

---

## 7. Шаг 6 — Получить уникальный список

1. Скопировать отфильтрованный столбец A на **Лист 2**
2. Скопировать отфильтрованный столбец B на Лист 2 (в следующий столбец)
3. Объединить оба столбца в один
4. **Данные → Удалить дубликаты**

Результат: чистый уникальный список всех тикеров, присутствующих на обеих биржах.

---

## 8. Шаг 7 — Сформировать синтетические пары

Используем функцию **СЦЕПИТЬ** для сборки строки в нужном формате TradingView.

**Формула (в столбце B):**
```excel
=СЦЕПИТЬ("BINANCE:"; A1; "/BYBIT:"; A1; ",")
```

**Результат для каждой строки:**
```
BINANCE:BTCUSDT/BYBIT:BTCUSDT,
BINANCE:ETHUSDT/BYBIT:ETHUSDT,
BINANCE:SOLUSDT/BYBIT:SOLUSDT,
...
```

> Запятая в конце каждой строки обязательна — это разделитель для импорта в TradingView.

Двойной клик по правому нижнему углу → протянуть до конца списка.

---

## 9. Шаг 8 — Импорт в TradingView

1. Скопировать весь столбец с синтетическими парами
2. Вставить в **Блокнот** (Notepad)
3. Сохранить файл как `synthetic_pairs.txt`
4. TradingView → Вотчлист → **Импортировать список символов**
5. Выбрать `synthetic_pairs.txt`

**Результат:** в вотчлисте появится список ~400 синтетических пар. Для каждой видны столбцы **«Объём»** и **«Изменение %»** — готовый скринер межбиржевых спредов.

## 10. Итоговый алгоритм (шпаргалка)

```
TradingView screener → Binance spot USDT → прокрутить всё → Ctrl+A → список "Binance"
TradingView screener → Bybit spot → прокрутить всё → Ctrl+A → список "Bybit"
Экспорт обоих списков → binance.txt, bybit.txt

Excel:
  Данные → Из текста → разделитель "запятая"
  Транспонировать строки → столбцы
  Ctrl+H: убрать "BINANCE:", убрать "BYBIT:"
  СЧЁТЕСЛИ для пересечения → фильтр → только единицы
  Лист 2 → удалить дубликаты → уникальный список

Формула: =СЦЕПИТЬ("BINANCE:"; A1; "/BYBIT:"; A1; ",")
Результат в Блокнот → сохранить → импорт в TradingView
```

## 11. Ключевые параметры

| Параметр | Значение |
|----------|---------|
| Тикеров Binance spot (USDT) | ~386 |
| Тикеров Bybit spot | ~465 |
| Общих тикеров (пересечение) | ~400 |
| Лимит загрузки TradingView | 150 (нужно прокрутить!) |
| Формат синтетической пары | `БИРЖА1:ТИКЕР/БИРЖА2:ТИКЕР,` |

---

## 12. Применение и ограничения

**Что показывает синтетическая пара:**
- Отношение спотовых цен двух бирж
- Отклонение от 1.0 = арбитражный спред
- Динамика спреда во времени на графике TradingView

**Ограничения:**
- Это визуальный инструмент, не алгоритмический скринер
- Не учитывает комиссии, проскальзывание, скорость исполнения
- Требует ручного мониторинга или настройки алертов в TradingView

**Следующий шаг:** заменить визуальный мониторинг TradingView на алгоритмический скринер через API (ccxt.pro + asyncio) — именно это и реализует наш бот.

## Глоссарий

| Термин | Что означает |
|--------|-------------|
| **Синтетическая пара** | Формула вида A/B, показывающая отношение цены актива A к активу B |
| **Скринер** | Инструмент фильтрации инструментов по заданным параметрам |
| **СЧЁТЕСЛИ** | Excel-функция: считает количество ячеек, удовлетворяющих условию |
| **СЦЕПИТЬ** | Excel-функция: объединяет текстовые строки (CONCATENATE в английской версии) |
| **Транспонировать** | Преобразование строки в столбец (и обратно) |
| **Пересечение списков** | Множество тикеров, присутствующих одновременно на обеих биржах |
