Отзывы по урокам

УРОК 11 Арбитраж на МОЕХ. Часть 1 — Введение в ТАТки ~19 мин

Первое введение в MOEX-арбитраж. Пара TATN/TATNP, построение графика спреда в TradingView, механика стратегии.

Ключевые параметры
  • TATN / TATNP — с 2020 одинаковые дивиденды → фундаментально одна бумага
  • TradingView: ratio (TATN÷TATNP=1) и difference (TATN−TATNP=0)
  • Прив тоньше по стакану → рвётся сильнее (до 1 рубля за прокидку)
  • 100 тыс. ₽ → ×10–15 за 1.5 мес. по словам учеников
Плюсы
  • Полностью рублёвая, без санкционных рисков
  • Низкий порог входа — 1 лот, всегда дадут исполнение
  • Налоговая эффективность: лонг+шорт = одно сальдо
  • Нет зависимости от крипто-инфраструктуры
Риски
  • Структурный сдвиг: дивиденды снова могут разойтись → спред изменится
  • При большом плече — маржин-колл
Вывод: Стратегия №1 для РФ. Подтверждена практикой. Идеальная точка старта для бота. MOEX ISS API бесплатен — мониторинг спреда легко автоматизировать.
УРОК 12 Арбитраж на МОЕХ. Часть 2 — Практика в стакане ~20 мин

Живая практика в Приводе. Настройка окон, два стакана, реальные входы и выходы.

Ключевые параметры
  • Прокидки прива: 1 000–2 000 лотов = до 1 рубля движения
  • Обычку сдвинуть: нужно 5 000–10 000 лотов
  • Выставлять лимитки за 0.5–1 рубль заранее
  • Escape = отменить все заявки (горячая клавиша)
  • Всегда лимитки (maker), не рынок (taker)
Плюсы
  • Прокидки — предсказуемые точки входа/выхода
  • Асимметрия ликвидности работает в пользу скальпера
Ограничения
  • Нужен опыт чтения стакана — не формализуется легко
  • Одну ногу закрыл — вторая может уйти против
Применимость в боте: Средняя. Прокидки сложно детектировать алгоритмически. Рекомендую сначала освоить руками, затем добавить в бот анализ объёма.
УРОК 13 Теория спреда и ТАТки — Два режима торговли ~25 мин

Фундаментальная теория статистического арбитража. История пары. Скальп vs среднесрочка.

История спреда
  • До 2019: 250 рублей (разные дивиденды)
  • После 2020: 5–20 рублей (дивиденды уравнены)
  • Февраль 2020: единственный выход ниже 0 (−6 руб.)
  • «Ноль» ТАТок = 5–6 рублей (там они бывают чаще всего)
Два торговых режима
  • Скальп: сигнал = скорость расхождения; 1–1000 лотов; частые сделки
  • Среднесрочка: набор неделями на минимуме; цель 5–20 руб.; 200–500 тыс. ₽
  • Усреднение разрешено — коинтеграция гарантирует возврат
Риски
  • «99 плюсовых сделок, одна отрицательная может порвать»
  • Шорт спреда → потенциально неограниченный убыток
Вывод: Обязательная теоретическая база перед автоматизацией. Оба режима можно закодить. Начать с алерт-бота по скальп-логике.
УРОК 14 Набор позиции и выход — Работа с крупными объёмами ~20 мин

Практика с позицией 6 250 лотов. Алерты, стратегии частичного закрытия, работа с прокидками.

Ключевые параметры
  • Алерт TradingView: 2.5 рубля (crossing) → на часы + телефон
  • Набирать частями: 250–1 000 лотов за раз
  • Заявки выхода расставлять заранее, не ждать движения
  • Комиссии: ×4 на круг (вход + выход × 2 ноги)
  • Прив закрывать первым (тоньше → лучше исполнение)
Применимость в боте
  • Алерт-функция — первое что реализовать
  • Логика «набирать частями при снижении» — поддаётся алгоритмизации
Вывод: Алерт по спреду — фаза 1 бота. Легко сделать через MOEX ISS API + Telegram.
УРОК 15 Комментарий по ТАТкам — Риски и реальный маржин-колл ~7 мин

Реальный случай маржин-колла. Потеря 40% депозита и восстановление.

Реальный инцидент
  • 3 августа: агрессивный шорт прива + параллельный арбитраж → маржин-колл
  • Потеря: −40% депозита при автоматическом закрытии
  • Автоматическое закрытие = сквиз = потери намного больше
  • Восстановление: на следующий день набрался снова, отбил ~половину
Правила риск-менеджмента
  • При приближении маржи → закрывать вручную, не ждать автозакрытия
  • Шорт спреда опаснее лонга (нет пола снизу)
  • Макс. 3 сделки в день — только на «хороших движениях»
  • Добавки скидывать на волатильности (0.3 руб.) → улучшать среднюю
🔴 Красный флаг для бота: обязательна система мониторинга маржи с Telegram-уведомлением при приближении к margin call level.
УРОК 16 Ответы на вопросы по ТАТкам — Q&A Длинный

Массовый Q&A по итогам первых недель. Инфраструктура, риски, психология.

Ключевые выводы
  • Готовых скринеров для MOEX-арбитража не существует → писать самому
  • Комиссии у всех пропов одинаковые — не конкурентный параметр
  • Вход: возле нуля. Выход: «когда увидишь достаточно денег»
  • Равное количество лотов на обе ноги (1:1) достаточно
  • Несколько пропов = диверсификация ✅
  • Усреднение допустимо (коинтеграция оправдывает)
Запрещено
  • «Бить в свою же заявку» с разных пропсчетов одного брокера — нарушение ЦБ РФ
Для бота: подтверждает необходимость самописного MOEX-скринера. Готовых решений нет — это наше конкурентное преимущество.
УРОК 17 Про фандинг и межбиржевой арбитраж — Следующий уровень ~5 мин

Итог 1-месячного фандинг-эксперимента. Анонс: золото MOEX vs форекс.

Фандинг-эксперимент
  • 15 000 USDT / 1 месяц → результат положительный
  • Стратегия Антона Ковалева: «нехер трогать, держишь и всё»
  • Если заёмные деньги под 8% → дополнительно ~30–40% сверху
Следующий этап (анонс)
  • Золото MOEX (фьючерс) vs форекс (спот) — до экспирации ~2% чистыми
  • Межбиржевой фьюч-спот: следующий раздел курса
  • Валютный арбитраж: уже часть основного дохода автора
Вывод: Фандинг — проще всего автоматизировать (нет стакана). Золото MOEX vs форекс — высокий потенциал, нужна отдельная инфраструктура.

Плюсы и риски всей стратегии

✅ ПЛЮСЫ
  • Доступна из РФ без ограничений — рублёвая, МОЕХ работает
  • Низкий входной порог — 100 тыс. ₽, 1 лот
  • Высокая повторяемость — подтверждена учениками
  • Налоговая эффективность — полное сальдирование
  • Нет крипто-инфраструктуры (VPN, иностранные биржи)
  • Коинтегрированная пара — усреднение обосновано
  • MOEX ISS API бесплатный и открытый
⚠ РИСКИ
  • Структурный сдвиг — если дивиденды снова разойдутся
  • Маржин-колл при агрессивном плече (−40% реальный случай)
  • Ликвидность ограничена при крупных объёмах (50+ тыс. лотов)
  • Нельзя «бить в свою заявку» (ЦБ)
  • Шорт спреда — потенциально неограниченный убыток
  • 99 плюс + 1 разрушительный минус = типичная арбитражная статистика

Архитектура бота: три фазы

Python MOEX ISS API

Фаза 1 — Мониторинг + Алерт (начать сейчас)

MOEX ISS API → Python → расчёт спреда → Telegram Bot → алерт трейдеру

Просто, надёжно. Ручное исполнение. DRY_RUN=true по умолчанию.

Фаза 2 — Полуавтомат (после освоения)

Сигнал → Подтверждение трейдера → Автовыставление лимитных заявок → Мониторинг

Брокерский API (CScalp или Finam). Нужно уточнить наличие API.

Фаза 3 — Полный автомат (после полного курса)

Сигнал → Автовход обе ноги → Мониторинг маржи → Алерт при ≥80% маржи → Автовыход

Прямая интеграция с брокерским API. Watchdog. Стоп по убытку.

Красные флаги — Что запрещено

Критично
🔴 Запускать бота на реальные деньги без DRY_RUN минимум 1–2 недели
🔴 Ждать автоматического маржин-колла — закрывать ВРУЧНУЮ при приближении
🔴 Брать шорт спреда агрессивно без учёта истории (спред был 250 рублей!)
🔴 «Бить в свою же заявку» на разных пропсчетах одного брокера — ЦБ РФ запрещает
