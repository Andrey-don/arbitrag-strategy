## Как добавить новый урок

1. Положить транскрибацию `.txt` в `для_notebooklm/` (имя: `01_тема.txt`)
2. Загрузить в NotebookLM, обработать промптом из `docs/LLM_обработка_уроков.md`
3. Сохранить конспект в `docs/модуль_01_арбитраж/урок_01_тема.md`
4. Обновить эту таблицу: ⏳ → ✅
5. git add → git commit → git push

## Этапы разработки бота

```
Этап 1: Изучение стратегии (текущий)
  └─ Обработать все транскрибации через NotebookLM
  └─ Заполнить docs/шаблоны_кода.md готовыми паттернами

Этап 2: Инфраструктура данных
  └─ src/data/ — подключение к биржам через ccxt
  └─ src/data/ — получение данных в реальном времени (websocket)

Этап 3: Детектор возможностей
  └─ src/strategies/ — алгоритм поиска арбитражного спреда
  └─ src/monitoring/ — мониторинг в реальном времени

Этап 4: Исполнение и риски
  └─ src/execution/ — выставление ордеров на двух биржах одновременно
  └─ src/risk/ — лимиты позиции, дневные потери, стоп-лосс

Этап 5: Запуск и мониторинг
  └─ DRY_RUN тест → реальный счёт
  └─ Telegram-уведомления о сделках
  └─ Дашборд с результатами
```
