# Обработка транскрибаций в NotebookLM

## Рекомендация: использовать NotebookLM (Google)

| Критерий | NotebookLM | ChatGPT |
|----------|-----------|---------|
| Загрузка файлов (.txt) | ✅ До 50 источников | ✅ Но ограничен по объёму |
| Контекстное окно | ✅ Очень большое (все транскрибации сразу) | ⚠️ Меньше |
| Токены не тратятся повторно | ✅ Источники хранятся в проекте | ❌ Каждый чат заново |
| Бесплатно | ✅ | ⚠️ GPT-4 платный |

**Стратегия работы с NotebookLM:**
1. Создать **один проект** в NotebookLM
2. Загрузить в него все `.txt` транскрибации из папки `для_notebooklm/`
3. Использовать промпт ниже для каждой транскрибации
4. Сохранять результат в `docs/модуль_01_арбитраж/`

Ты — помощник по созданию документации для проекта торгового бота на Python по стратегии арбитража.

Я дам тебе транскрибацию видео опытного трейдера (15+ лет торговли по арбитражу). Транскрибация сделана автоматически, поэтому может содержать ошибки — исправляй их по контексту.

ТВОЯ ЗАДАЧА — создать структурированный markdown-документ по следующей схеме:

---
# Урок [номер]: [название темы]

## 1. Краткое резюме (3–5 предложений)
Что разбирается в этом видео. Зачем это важно для арбитражного бота.

## 2. Ключевые концепции арбитража
Список концептов с кратким объяснением (буллет-пойнты).
Особо выделить: типы арбитража, условия входа/выхода, управление рисками.

## 3. Алгоритм стратегии
Пошаговое описание действий трейдера в сделке.
Формализовать в виде псевдокода или блок-схемы.

## 4. Параметры стратегии
Таблица параметров: название, значение, для чего используется.
(спред, размер позиции, таймфрейм, биржи, инструменты и т.д.)

## 5. Управление рисками
Стоп-лоссы, максимальная просадка, риск на сделку.
Как трейдер управляет позицией.

## 6. Важные замечания
Подводные камни, нюансы, которые трейдер особо подчёркивает.
Типичные ошибки начинающих арбитражников.

## 7. Применение в боте (реализация)
Как конкретно автоматизировать эту логику в Python.
Какие библиотеки использовать (ccxt, asyncio, websockets).
Псевдокод или скелет кода для этой части стратегии.

## Итоговый граф знаний

После обработки всех транскрибаций — построить граф через `/graphify` в Claude Code:
1. Скопировать все markdown-документы в одну папку
2. Запустить `/graphify` — создаст HTML + JSON граф связей
3. Граф покажет структуру стратегии: Обнаружение → Вход → Управление позицией → Выход → Риск
