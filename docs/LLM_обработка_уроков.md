# Обработка транскрибаций в NotebookLM

## Рекомендация: использовать NotebookLM (Google)

| Критерий | NotebookLM | ChatGPT |
|----------|-----------|---------|
| Загрузка .txt файлов | ✅ До 50 источников | ✅ Но ограничен по объёму |
| Контекстное окно | ✅ Огромное (весь курс сразу) | ⚠️ Меньше |
| Токены НЕ тратятся повторно | ✅ Источники хранятся в проекте | ❌ Каждый чат заново |
| Бесплатно | ✅ | ⚠️ GPT-4 платный |

**Стратегия работы:**
1. Создать **один проект** в NotebookLM: https://notebooklm.google.com/
2. Загрузить все `.txt` транскрибации из папки `для_notebooklm/`
3. Обработать каждую транскрибацию промптом ниже
4. Сохранить результат в `docs/модуль_01_арбитраж/урок_NN_тема.md`
5. Обновить статус в `docs/карта_курса.md`

## Промпт для обработки каждой транскрибации

Скопируй в NotebookLM, вставь транскрибацию в конец:

---

```
Ты — помощник по созданию документации для проекта торгового бота на Python по стратегии арбитража.

Я дам тебе транскрибацию видео опытного трейдера (15+ лет арбитражной торговли). Транскрибация автоматическая — содержит ошибки OCR, исправляй по контексту.

ЗАДАЧА — создать структурированный markdown-документ:

---
# Урок [номер]: [название темы]

## 1. Краткое резюме (3–5 предложений)
Что разбирается в этом видео. Зачем это важно для арбитражного бота.

## 2. Ключевые концепции арбитража
Список концептов с объяснением (буллет-пойнты).
Выделить: тип арбитража, условия входа/выхода, инструменты, биржи.

## 3. Алгоритм стратегии
Пошаговое описание действий трейдера.
Оформить как нумерованный список или псевдокод.

## 4. Параметры стратегии
Таблица: название параметра | значение/диапазон | для чего используется
(спред, размер позиции, таймфрейм, комиссии, и т.д.)

## 5. Управление рисками
Стоп-лоссы, максимальный размер позиции, дневные лимиты.
Как трейдер управляет позицией при неблагоприятном движении.

## 6. Важные замечания
Подводные камни и нюансы, которые трейдер особо подчёркивает.
Типичные ошибки начинающих арбитражников.

## 7. Реализация в боте (Python)
Как автоматизировать эту логику.
Псевдокод или скелет кода (ccxt, asyncio, websockets).
Что нужно реализовать в src/ проекта.
