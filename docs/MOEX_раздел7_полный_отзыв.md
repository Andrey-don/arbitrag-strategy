## Ядро стратегии: ТАТки

### Суть
**ТАТНн (обычка) / ТАТНп (прив)** — статистически коинтегрированная пара акций Татнефти. С 2020 года платят одинаковые дивиденды → теоретически должны стоить одинаково. На практике — постоянно расходятся и сходятся.

### Параметры стратегии
| Параметр | Значение |
|---------|--------|
| Инструмент | TATN / TATNP (МОЕХ, секция ММВБ) |
| Типичный спред | 2–5 рублей (скальпинг) |
| Исторический максимум | 250 рублей (до 2020) / 20 рублей (после 2020) |
| Пол спреда | ~0 рублей (ниже не ходит) |
| Режим 1 (скальпинг) | Ловить раздвижки 1–3 рубля, 1–1000 лотов |
| Режим 2 (среднесрочка) | Набрать 10–50 тыс. лотов, ждать 5–20 руб |
| Целевая прибыль (скальпер) | ×10–15 от начального капитала за 1.5 мес. |
| Целевая прибыль (среднесрочник) | 200–500 тыс. ₽ на позиции 40–50 тыс. лотов |
| Плечо | Не указано явно (через пропсчёт — встроенное) |
| Налоги | Полностью сальдируются (лонг+шорт = одна позиция) |
| Брокер | CScalp (Коскэп) — рекомендован |
| Терминал | Привод (от CScalp) |

## Плюсы стратегии

### 1. Доступна из РФ без ограничений ✅
- МОЕХ работает нормально
- Рублёвая торговля — никаких валютных рисков и санкций
- Не нужен VPN, нет риска заморозки активов

### 2. Низкий входной порог ✅
- 100 тыс. ₽ — рекомендуемый минимум
- С 1 лота (минимум) уже зарабатываешь
- Ликвидность всегда даёт исполнение при малых объёмах

### 3. Высокая повторяемость ✅
- «Слишком легко» — цитата автора
- Большинство учеников получают результат
- Множество участников в группе раскачали депо ×10–15 за месяц
- «99 плюсовых сделок» — статистически выигрышная стратегия

### 4. Налоговая эффективность ✅
- Прибыль и убыток по двум ногам сальдируются
- Нет ситуации: «здесь гигантский минус, там гигантский плюс, а налог с плюса»
- Только чистый результат облагается налогом

### 5. Нет зависимости от крипто-инфраструктуры ✅
- Не нужны иностранные биржи
- Не нужны USDT, стейблкоины
- Не нужны P2P операции
- Полностью легально

### 6. Коинтегрированная пара (статистическое преимущество) ✅
- Спред не может уходить ниже 0 (исторически)
- Всегда возвращается → усреднение работает
- Оба инструмента физически одна компания → фундаментально привязаны

## Риски стратегии

### 1. Структурный сдвиг спреда 🔴
- До 2020: спред 250 рублей → теперь 5–20 рублей
- Причина: уравнивание дивидендов изменило фундаментальную картину
- **Риск**: если снова изменится структура (дивиденды, buyback) — спред может сжаться до 0 навсегда или уйти ниже
- Вероятность: низкая, но не нулевая (см. Q&A: «16 лет на больших спредах»)

### 2. Маржин-колл при неправильном плече 🔴
- Автор потерял 40% депозита одним маржин-коллом
- При автоматическом закрытии — сильный сквиз
- **Решение**: всегда закрывать вручную если маржа приближается
- Шорт спреда (шорт обычки) опаснее лонга — нет ограничения сверху

### 3. Ликвидность при росте объёма 🟡
- До 1 000 лотов: отлично, всегда нальют
- 5 000+ лотов: заметно сложнее
- 50 000 лотов: набирать неделями, выходить часами
- **Для бота**: ограничения на размер позиции критичны

### 4. Возможность рыночной манипуляции 🟡
- Крупные участники «прокидывают» стакан специально
- Могут раздвинуть спред в невыгодную сторону
- Без стакана и чутья сложно отличить сигнал от шума

### 5. Мониторинг (почти full-time) 🟡
- Среднесрочник может поставить алерт и уйти
- Скальпер должен сидеть у экрана
- Автоматизация необходима для масштабирования

### 6. Запретная зона: «бить в свою же заявку» 🔴
- Разные пропсчета на одной бирже — нельзя торговать против самого себя
- ЦБ РФ следит, могут вызвать «на ковёр»

## Потенциал автоматизации (бот)

### Что автоматизировать можно ✅

| Функция | Реализация |
|--------|-----------|
| Мониторинг спреда | REST API MOEX (iss.moex.com) — бесплатно |
| Сигнал при отклонении | Python: цена A / цена B → отклонение |
| Алерт в Telegram | python-telegram-bot |
| Автоматический ввод заявок | MOEX ISS API или брокерский API |
| Расчёт комиссий | Фиксированная формула |
| P&L учёт | Простая арифметика |

### Ограничения для бота ⚠️

| Ограничение | Детали |
|------------|-------|
| Брокерский API | Не все пропы дают API. CScalp — проверить |
| Стакан в реальном времени | MOEX WebSocket (ISS + WebSocket addon) |
| Скальп на 1 лоте | Возможно через API |
| Среднесрочка | Проще автоматизировать |
| Решение по стакану | Сложно — нужен анализ уровней |

### Рекомендуемая архитектура бота

```python
# Логика скальп-мониторинга
while True:
    price_tatn = get_price("TATN")
    price_tatnp = get_price("TATNP")
    spread = price_tatn - price_tatnp
    
    if spread < THRESHOLD_LOW:      # напр. < 0.5 руб
        alert("ТАТки: лонг обычки + шорт прива")
        if AUTO_MODE:
            place_order("TATN", "buy", lots)
            place_order("TATNP", "sell", lots)
    
    elif spread > THRESHOLD_HIGH:   # напр. > 3 руб
        alert("ТАТки: схождение близко, можно закрываться")
    
    sleep(1)
```

### Брокерский API MOEX
- **MOEX ISS API**: бесплатный, REST, данные котировок в реальном времени
- **Finam Trade API**: Finam поддерживает API для автоматизации (проверить условия)
- **Альтернатива**: библиотека `moexalgo` (Python) — обёртка над ISS
- **CScalp API**: уточнить у поддержки (@KasCapital_Robot)
