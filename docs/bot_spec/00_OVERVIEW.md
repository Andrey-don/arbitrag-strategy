# MOEX Arbitrage Trading System — Обзор и Философия

## Официальное название системы

**MOEX Arbitrage Trading System** (сокращённо: MATS)

### Почему не «бот» и не «агент»

| Термин | Проблема |
|--------|---------|
| «Бот» | Подразумевает одну простую программу. У нас несколько компонентов |
| «AI-агент» | Подразумевает LLM в критическом пути. Торговое ядро — детерминированное |
| «Агент» без уточнения | Размытый термин, путаница |

**Правильно:** система состоит из нескольких компонентов. Каждый компонент — либо **детерминированный сервис** (бот-логика), либо **агент** (автономный, может использовать LLM). Координирует их **оркестратор**.

## Три типа логики — три разных подхода

### Тип 1: Правило (Rule) — код, не меняется без явного решения

```python
# Это правило — зашито в код
if spread_rub < 0:
    signal = Signal.LONG_SPREAD
```

Источник: урок, backtesting, явное решение команды.
Менять только осознанно, с обновлением bot_spec.

### Тип 2: Порог (Config) — число в конфиге, меняется по данным

```python
# Это параметр — в .env или конфиге
SPREAD_ALERT_RUB = 2.5       # из урок_14
MARGIN_CRITICAL_PCT = 0.65   # из урок_15
MAX_ONE_LEG_SECONDS = 5      # TODO: уточнить
```

Меняется на основе опыта торговли без правки кода.

### Тип 3: Решение агента (Agent Decision) — LLM или человек

```
# Это НЕ в критическом пути:
"Сегодня волатильность аномально высокая, рекомендую
 снизить объём позиции на 50% до конца дня."
```

LLM/агент работает async, результат — рекомендация, не приказ.
Торговое ядро не ждёт LLM. LLM не блокирует исполнение.

## Философия

### 1. Production-first с первого дня
Каждый урок разбирается с вопросом: «Что произойдёт, если это упадёт в 3 ночи?»

### 2. LLM — не в критическом пути
Торговое ядро: детерминированный код. LLM: только анализ и отчёты, async.

### 3. Правило / Порог / Решение — три разные вещи
Не смешивать. Правило — в коде. Порог — в конфиге. Решение агента — рекомендация.

### 4. Нет «временных решений» в production
Любой обход без failsafe — технический долг. Лучше не реализовать фичу.

### 5. Экономия токенов = скорость работы
Читать только один файл bot_spec при работе над компонентом, не весь проект.
