## Сигнал входа (ТАТки)

```
Спред = TATN - TATNP  (или ratio = TATN / TATNP)
```

| Условие | Действие |
|---------|---------|
| Спред < 0 рублей (или ~0) | Лонг спреда: ЛОНГ TATN + ШОРТ TATNP |
| Спред > 5–7 рублей (скальп) | Шорт спреда: ШОРТ TATN + ЛОНГ TATNP |
| Спред > 2.5 руб (алерт) | Уведомление → оценить вход |

**Правило направления (из урок_12, 13:30):**
- На схождение: ШОРТ обычку (TATN) + ЛОНГ префы (TATNP)
- На расхождение: ШОРТ префы (TATNP) + ЛОНГ обычки (TATN)

### Почему лонг спреда безопаснее шорта

Исторически спред не уходит ниже 0 (кроме ковид фев.2020 = −6 руб., разовый выброс).
Максимум вверх: ~250 руб (до 2020), ~20 руб (сент. 2023), обычно 5–20 руб.
→ **Лонг спреда**: риск ограничен снизу (~0).
→ **Шорт спреда**: риск теоретически неограничен вверх.

### Зоны набора позиции (среднесрочка)

| Спред | Действие |
|-------|---------|
| −2…0 руб | Активный набор лонга спреда (лучшая зона) |
| 0…2 руб | Набор частями (основная зона, «у единицы») |
| 2…5 руб | Удержание / частичное закрытие добавок |
| 5+ руб | Закрытие основной позиции лесенкой |
| 10+ руб | Агрессивное закрытие |

## Правила исполнения

### Вход
1. Ставить лимитку в уровень — «накликать», не бить по рынку
2. Набирать малыми частями (250, 500, 1 000 лотов)
3. Расставлять заявки заранее на 0.5–1 рубль от текущей цены
4. При прокидке прива — использовать для улучшения средней

### Выход
1. Ставить заявки заранее на прогнозируемых уровнях расхождения
2. Закрывать ОБЕ ноги одновременно насколько возможно
3. При быстром расхождении (айсы) — выйти за 20 сек через стакан
4. При медленном — разгружаться за час лимитками

### Комиссии
- 4 комиссии на полный круг: вход нога 1 + вход нога 2 + выход нога 1 + выход нога 2
- Всегда лимитки → меньше комиссия (maker vs taker)

## Налоги

Лонг TATN + Шорт TATNP = **полное сальдирование**.
Налог берётся только с чистого P&L пары, не с каждой ноги отдельно.
Это ключевое преимущество перед обычным скальпингом.

## Пороги входа/выхода ТАТки в % (TradingView + стакан)

> Применимо когда оба инструмента наложены на одном графике TradingView в режиме %.
> «Расстояние между линиями» = визуальный спред.

### Как читать график

Режим % в TradingView показывает **относительное изменение от начала периода**, а не абсолютный спред в рублях. Расстояние между линиями — это и есть торгуемый спред.

```
Оранжевый (TATNP) НИЖЕ синего (TATN):
  → Префы упали сильнее / выросли меньше
  → Действие: ЛОНГ TATN + ШОРТ TATNP
  → Ждём: оранжевая линия поднимется к синей

Синий (TATN) НИЖЕ оранжевого (TATNP):
  → Обычка упала сильнее / выросла меньше
  → Действие: ШОРТ TATN + ЛОНГ TATNP
  → Ждём: синяя линия поднимется к оранжевой
```

### Перевод % в рубли (при ценах TATN/TATNP ≈ 600–650 руб)

| Расхождение на графике % | Абсолютный спред (руб) | Оценка |
|--------------------------|----------------------|--------|
| 0.1–0.3% | ≤ 2 руб | Не интересно — не входить |
| **~0.4%** | **~2.5 руб** | **Алерт — начать наблюдать** |
| **~0.7–0.8%** | **~5 руб** | **Вход скальп (лимитный ордер)** |
| **~1.0–1.5%** | **~7–10 руб** | **Хороший вход** |
| >1.5% | >10 руб | Среднесрочная позиция |

### Учёт комиссий (полный круг = 4 комиссии)

```
Вход нога 1 + Вход нога 2 + Выход нога 1 + Выход нога 2

При тарифе 0.05%/сторону:  4 × 0.05% = 0.20% на круг
При тарифе 0.10%/сторону:  4 × 0.10% = 0.40% на круг

→ Минимальная разница для прибыли: 0.3–0.5%
→ Входить при разнице < 0.3% — комиссии съедают всю прибыль
```

### Пороги входа/выхода (конфиг)

```python
# --- ТАТки: пороги в % (TradingView-совместимые) ---
TATKI_SPREAD_ALERT_PCT   = 0.40   # % — начать наблюдать, готовить ордера
TATKI_SPREAD_ENTRY_PCT   = 0.70   # % — вход скальп (лимитный ордер)
TATKI_SPREAD_ENTRY_GOOD  = 1.00   # % — хороший вход
TATKI_SPREAD_MID_TERM    = 1.50   # % — среднесрочная позиция
TATKI_SPREAD_TARGET_PCT  = 0.10   # % — выход (линии сошлись, цель достигнута)
TATKI_COMMISSION_ROUND   = 0.20   # % — минимальный порог для прибыли (тариф 0.05%)

# --- Абсолютные уровни (руб) при ценах ~630 руб ---
TATKI_ALERT_RUB   = 2.5   # руб — алерт
TATKI_ENTRY_RUB   = 5.0   # руб — скальп-вход
TATKI_TARGET_RUB  = 0.5   # руб — выход (схождение)
```

### Когда выходить из позиции

| Ситуация | Действие |
|---------|---------|
| Линии сошлись (разница < 0.1–0.2%) | Закрыть обе ноги — цель достигнута |
| Скальп: расхождение выросло на 5–7 руб (~0.8%) | Закрываться, не ждать больше |
| Среднесрочка: расхождение 10–20 руб (~1.5–3%) | Закрывать лесенкой (частями) |
| Участника нет 2–3 дня, вкат не пришёл | Закрыться в убыток ≤ 2% (правило Каширина) |

### Важно: только лимитные ордера

- Вход: **ТОЛЬКО лимитный** (maker) — не бить по рынку
- Выход при схождении: лимитный (расставить заранее)
- Выход экстренный (маржа, участник исчез): маркет допускается

## Три графика для принятия решений (по методу Клевцова)

Одного графика недостаточно. Каждый из трёх отвечает на свой вопрос.

| График | Формула | Отвечает на вопрос |
|--------|---------|-------------------|
| **Наложение %** | TATN + TATNP на одном экране | *Куда* расходятся прямо сейчас |
| **Разница** | `TATN − TATNP` (руб) | *Насколько* расошлись, где исторические уровни |
| **Ratio** | `TATN / TATNP` | *Норма или отклонение* — нормализованный сигнал |

### Рабочая схема (последовательность анализа)

```
1. Ratio      → сигнал:        "отклонение от нормы есть"
2. Разница    → подтверждение: "в рублях это X руб, уровень интересный"
3. Наложение  → направление:   "TATNP упал — значит лонг спреда"
```

### График 1 — Наложение (%)

**Как читать:**
```
Оранжевый (TATNP) НИЖЕ синего (TATN)
  → TATNP упал сильнее → ЛОНГ TATN + ШОРТ TATNP

Синий (TATN) НИЖЕ оранжевого (TATNP)
  → TATN упал сильнее → ШОРТ TATN + ЛОНГ TATNP
```

**Ограничение:** не показывает размер в рублях и не показывает историческую норму. Меняется при смене периода графика.

### График 2 — Разница (TATN − TATNP, рубли)

**Настройка TradingView:** ввести в строке инструмента `TATN - TATNP`

**Ключевые горизонтальные уровни:**
```
0 руб   — исторический пол (ниже практически не уходит; ковид 2020: −6 руб, разовый выброс)
2.5 руб — алерт, начинать наблюдать
5 руб   — вход скальп
7 руб   — хороший вход
10 руб  — добавка к среднесрочной позиции
20 руб  — зона агрессивного выхода
```

**Это основной рабочий график.** Без него нельзя выставить конкретные пороги в рублях.

### График 3 — Ratio (TATN / TATNP)

**Настройка TradingView:** ввести `TATN / TATNP`

**Ключевые горизонтальные уровни:**
```
< 1.000 — TATNP дороже TATN → сильный сигнал лонга спреда (редкость)
1.000   — паритет
1.003   — алерт (~2 руб при цене 630)
1.008   — вход скальп (~5 руб)
1.011   — хороший вход (~7 руб)
1.016+  — среднесрочная позиция (10+ руб)
```

**Преимущество:** нормализован — не «плывёт» при общем росте или падении акций. Лучший для автоматического алерта бота.

### Настройка TradingView (три панели на одном экране)

```
Панель 1 (верх):    TATN + TATNP наложение в режиме %
Панель 2 (середина): TATN - TATNP  (разница в руб, уровни 0/2.5/5/7/10/20)
Панель 3 (низ):     TATN / TATNP  (ratio, уровни 1.000/1.003/1.008/1.011)
```

### Реализация в боте MATS

```python
def calc_spread_signals(price_tatn: float, price_tatnp: float) -> dict:
    spread_rub = price_tatn - price_tatnp       # разница (руб)
    ratio      = price_tatn / price_tatnp       # деление
    spread_pct = (ratio - 1) * 100              # отклонение от нормы (%)

    signal = "NO_SIGNAL"
    if ratio < 1.000:
        signal = "LONG_SPREAD_STRONG"   # TATNP дороже — редкость, сильный сигнал
    elif spread_rub >= 7.0 and ratio >= 1.011:
        signal = "LONG_SPREAD_GOOD"     # хороший вход
    elif spread_rub >= 5.0 and ratio >= 1.008:
        signal = "LONG_SPREAD_SCALP"    # скальп-вход
    elif spread_rub >= 2.5:
        signal = "ALERT_WATCH"          # наблюдать, готовить ордера
    elif spread_rub <= 0.5:
        signal = "EXIT_TARGET"          # цель достигнута, закрывать

    return {
        "spread_rub": spread_rub,
        "ratio":      ratio,
        "spread_pct": spread_pct,
        "signal":     signal,
    }
```

## Валютный арбитраж Si / Юань / Форекс (источник: р9-12, Каширин)

### Правило хеджирования Форекса

Не брать 100% хедж на Форексе — это осознанная стратегия, не упущение.

| Ситуация | % хеджа на Форексе |
|---------|-------------------|
| Базовый (по умолчанию) | **70–80%** |
| Внутри дня + сильный участник на ММВБ | **50–60%** |
| Перенос через ночь | добрать до **80%** |
| Перед новостями (если уже в позиции) | **80%** обязательно |

**Логика неполного хеджа:** если ММВБ двигается благодаря участнику, а Форекс отстаёт — позиция с 80% хеджем даёт бо́льший P&L, чем со 100%. При 100% хедже прибыль ограничена только изначальной раздвижкой.

```python
FOREX_HEDGE_PCT_DEFAULT     = 0.80   # базовый хедж
FOREX_HEDGE_PCT_INTRADAY    = 0.60   # внутри дня при сильном участнике
FOREX_HEDGE_PCT_OVERNIGHT   = 0.80   # перенос через ночь
FOREX_HEDGE_PCT_NEWS        = 0.80   # перед новостями (обязательно)
```

### Правило работы от участника (айсберги)

```
Участник (крупные айсберги) активен → держать позицию
Участник исчез → наблюдать 2–3 рабочих дня
Если через 2–3 дня нет участника и нет вката → закрываться (убыток ≤ 2%)
Причина: "цена стала эффективной, схлопываться нечему"
```

- Перезаход: если участник появился снова после закрытия → открыться повторно
- Айсберги = основной сигнал для Si/юань/Форекс стратегии

```python
PARTICIPANT_WATCH_DAYS = 3       # дней ожидания участника перед закрытием
PARTICIPANT_EXIT_LOSS_MAX = 0.02 # максимальный убыток при закрытии "без участника"
```
