## Сигнал входа (ТАТки)

```
Спред = TATN - TATNP  (или ratio = TATN / TATNP)
```

| Условие | Действие |
|---------|---------|
| Спред < 0 рублей (или ~0) | Лонг спреда: ЛОНГ TATN + ШОРТ TATNP |
| Спред > 5–7 рублей (скальп) | Шорт спреда: ШОРТ TATN + ЛОНГ TATNP |
| Спред > 2.5 руб (алерт) | Уведомление → оценить вход |

**Правило направления (из урок_12, 13:30):**
- На схождение: ШОРТ обычку (TATN) + ЛОНГ префы (TATNP)
- На расхождение: ШОРТ префы (TATNP) + ЛОНГ обычки (TATN)

### Почему лонг спреда безопаснее шорта

Исторически спред не уходит ниже 0 (кроме ковид фев.2020 = −6 руб., разовый выброс).
Максимум вверх: ~250 руб (до 2020), ~20 руб (сент. 2023), обычно 5–20 руб.
→ **Лонг спреда**: риск ограничен снизу (~0).
→ **Шорт спреда**: риск теоретически неограничен вверх.

### Зоны набора позиции (среднесрочка)

| Спред | Действие |
|-------|---------|
| −2…0 руб | Активный набор лонга спреда (лучшая зона) |
| 0…2 руб | Набор частями (основная зона, «у единицы») |
| 2…5 руб | Удержание / частичное закрытие добавок |
| 5+ руб | Закрытие основной позиции лесенкой |
| 10+ руб | Агрессивное закрытие |

## Правила исполнения

### Вход
1. Ставить лимитку в уровень — «накликать», не бить по рынку
2. Набирать малыми частями (250, 500, 1 000 лотов)
3. Расставлять заявки заранее на 0.5–1 рубль от текущей цены
4. При прокидке прива — использовать для улучшения средней

### Выход
1. Ставить заявки заранее на прогнозируемых уровнях расхождения
2. Закрывать ОБЕ ноги одновременно насколько возможно
3. При быстром расхождении (айсы) — выйти за 20 сек через стакан
4. При медленном — разгружаться за час лимитками

### Комиссии
- 4 комиссии на полный круг: вход нога 1 + вход нога 2 + выход нога 1 + выход нога 2
- Всегда лимитки → меньше комиссия (maker vs taker)

## Налоги

Лонг TATN + Шорт TATNP = **полное сальдирование**.
Налог берётся только с чистого P&L пары, не с каждой ноги отдельно.
Это ключевое преимущество перед обычным скальпингом.

## Валютный арбитраж Si / Юань / Форекс (источник: р9-12, Каширин)

### Правило хеджирования Форекса

Не брать 100% хедж на Форексе — это осознанная стратегия, не упущение.

| Ситуация | % хеджа на Форексе |
|---------|-------------------|
| Базовый (по умолчанию) | **70–80%** |
| Внутри дня + сильный участник на ММВБ | **50–60%** |
| Перенос через ночь | добрать до **80%** |
| Перед новостями (если уже в позиции) | **80%** обязательно |

**Логика неполного хеджа:** если ММВБ двигается благодаря участнику, а Форекс отстаёт — позиция с 80% хеджем даёт бо́льший P&L, чем со 100%. При 100% хедже прибыль ограничена только изначальной раздвижкой.

```python
FOREX_HEDGE_PCT_DEFAULT     = 0.80   # базовый хедж
FOREX_HEDGE_PCT_INTRADAY    = 0.60   # внутри дня при сильном участнике
FOREX_HEDGE_PCT_OVERNIGHT   = 0.80   # перенос через ночь
FOREX_HEDGE_PCT_NEWS        = 0.80   # перед новостями (обязательно)
```

### Правило работы от участника (айсберги)

```
Участник (крупные айсберги) активен → держать позицию
Участник исчез → наблюдать 2–3 рабочих дня
Если через 2–3 дня нет участника и нет вката → закрываться (убыток ≤ 2%)
Причина: "цена стала эффективной, схлопываться нечему"
```

- Перезаход: если участник появился снова после закрытия → открыться повторно
- Айсберги = основной сигнал для Si/юань/Форекс стратегии

```python
PARTICIPANT_WATCH_DAYS = 3       # дней ожидания участника перед закрытием
PARTICIPANT_EXIT_LOSS_MAX = 0.02 # максимальный убыток при закрытии "без участника"
```
