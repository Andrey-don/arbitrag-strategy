### Режим 1 — Ручной (доступен сейчас)

**Схема:**
```
TradingView алерт (ratio > 1.008 или разница > 5 руб)
  → смотришь на три графика (наложение / разница / ratio)
  → оцениваешь стакан в CScalp
  → ставишь лимитные ордера на обе ноги вручную
  → ждёшь схождения → закрываешь вручную
```

**Что нужно:**

| Компонент | Статус | Действие |
|-----------|--------|---------|
| TradingView (три графика) | ✅ описано | Настроить за 30 минут |
| TradingView алерты | ✅ встроено | Настроить crossing 2.5 руб / ratio 1.008 |
| Счёт у брокера / KasCapital | ❓ | Зарегистрироваться: kas.wtf/referal/KlevcovDIV |
| CScalp / Привод | ❓ | Установить, подключить к счёту |
| Капитал | ❓ | Минимум 100 000 ₽ (ТАТки) |

**Вывод:** торговать можно уже сейчас, без единой строки кода.

### Режим 2 — Полуавтоматический (алерт → ручное исполнение)

**Схема:**
```
Python скринер → MOEX ISS API → расчёт spread / ratio
  → Telegram-сообщение с числами и сигналом
  → трейдер видит алерт → открывает CScalp → вводит ордера вручную
```

**Что нужно написать (фаза 1, ~1–2 недели):**

| Компонент | Объём | Сложность |
|-----------|-------|----------|
| MOEX ISS API — опрос цен | ~50 строк | Низкая (API бесплатный) |
| Расчёт spread / ratio / сигнала | ~30 строк | Тривиально |
| Telegram-бот (алерты) | ~50 строк | Низкая |
| Итого | ~200 строк | **Фаза 1** |

**Где запускать скринер:**

| Вариант | Подходит? | Примечание |
|---------|----------|-----------|
| Локальный ПК | ✅ достаточно | ПК должен работать пока идёт торговля |
| VPS в России | ✅ надёжнее | 300–500 ₽/мес, работает 24/7 |

Для фазы 1 хватает локального ПК — скринер не торгует, только уведомляет.

### Режим 3 — Автоматический (полный бот)

**Ключевая проблема: CScalp не имеет публичного API.**

CScalp/Привод — торговый терминал, не брокер. Автоматически выставлять ордера через него нельзя. Для автоторговли нужен **брокер с открытым API**.

**Брокеры с API для MOEX акций:**

| Брокер | API | Python SDK | Акции ММВБ |
|--------|-----|-----------|-----------|
| **T-Invest (Тинькофф)** | ✅ открытый | ✅ официальный | ✅ |
| **ФИНАМ Trade API** | ✅ | ✅ | ✅ |
| **Алор-Брокер** | ✅ ALOR API | ✅ | ✅ |
| **БКС** | ✅ | ⚠️ неофициальный | ✅ |
| **KasCapital (проп)** | ❓ уточнить | ❌ | ❓ |

> **Критично решить первым:** если торгуете через KasCapital проп-счёт — уточнить наличие API. Если API нет, автоматическое исполнение через них невозможно. Потребуется параллельный счёт у брокера с API.

**Что нужно написать (фаза 3–4, ~2–3 месяца):**

```
src/
├── data/         ← MOEX ISS API, опрос цен каждые 500 мс
├── strategies/   ← calc_spread_signals() → решение входить/нет
├── risk/         ← проверка маржи, лимитов, дневного стопа
├── execution/    ← выставление лимитных ордеров через брокерский API
├── monitoring/   ← Telegram-алерты, логирование
└── utils/        ← конфиг, .env
```

**Где запускать:**

| Вариант | Подходит? | Требования |
|---------|----------|-----------|
| Локальный ПК | ⚠️ рискованно | UPS, резервный интернет, не выключать |
| **VPS в России** | ✅ рекомендуется | 500–1 000 ₽/мес, низкая задержка до MOEX |
| Облако (Yandex Cloud) | ✅ | Надёжно, но дороже |

## Среда выполнения

### Требования к latency
- Для среднесрочки ТАТки: latency не критична (минуты/часы горизонт)
- Для скальпинга: важна, но co-location не нужен (в отличие от HFT)
- DRY_RUN=true по умолчанию везде до явного разрешения

### Рекомендованное окружение (производство)
- VPS в России (минимизировать задержку до MOEX)
- Или локальная машина с надёжным интернетом
- UPS / резервное интернет-соединение (для production фазы)
