## Расчёт риска на сделку (источник: р9-09, Виктор)

### Формула

```
Просадка фропа = 20% депозита
Риск на 1 сделку = Просадка / N_инструментов
```

| Депозит | Просадка 20% | N инструментов | Риск на сделку |
|--------|-------------|---------------|----------------|
| 300 000 ₽ | 60 000 ₽ | 5 | **12 000 ₽** |
| 300 000 ₽ | 60 000 ₽ | 10 | **6 000 ₽** |
| 100 000 ₽ | 20 000 ₽ | 5 | **4 000 ₽** |

**Риск в процентах от депозита:** ~1% на сделку (при 5 инструментах).

### Параметры конфига

```python
DRAWDOWN_LIMIT_PCT = 0.20        # 20% — стандартная просадка фропа
N_INSTRUMENTS_DIVERSIFICATION = 5  # минимум инструментов
RISK_PER_TRADE_PCT = DRAWDOWN_LIMIT_PCT / N_INSTRUMENTS_DIVERSIFICATION  # = 4%
# Но в денежном выражении: RISK_PER_TRADE_RUB = DEPOSIT * 0.04 (при N=5)
# Корректнее: RISK_PER_TRADE_RUB = DEPOSIT * DRAWDOWN_LIMIT_PCT / N_INSTRUMENTS
```

### Инструменты для диверсификации (пример)
- Евро / Юань (вечники)
- Доллар / Юань (ЕД/Юань форекс)
- Золото / Серебро
- Шишка / Юань (акция нефти vs юань)
- ТАТки (TATN / TATNP)

## Расчёт лотности (источник: р9-09, Виктор)

**Алгоритм (один раз в неделю / при смене инструмента):**

1. Открыть дневной график инструмента
2. Взять **5 средних свечей** (убрать максимальную и минимальную)
3. Определить среднее дневное движение → принять за **1% риска**
4. Подобрать лотность так: если цена пройдёт это движение по одной ноге → убыток = RISK_PER_TRADE_RUB

```python
def calc_lot_size(price_a, price_b, avg_daily_move_pct, risk_rub):
    """
    Рассчитать лотность пары (A, B) при заданном риске.
    avg_daily_move_pct: средний дневной ход как доля (например, 0.01 = 1%)
    """
    # Стоимость 1 лота инструмента A при движении на avg_daily_move_pct
    loss_per_lot_a = price_a * avg_daily_move_pct
    lots_a = risk_rub / loss_per_lot_a
    # Лотность B подбирается так, чтобы позиция была нейтральной по стоимости
    lots_b = lots_a * price_a / price_b
    return int(lots_a), int(lots_b)
```

**Пример — Шишка / Юань (Виктор):**
```
Риск на сделку: 12 000 ₽
Среднее движение: 1%
→ 13 лотов Шишки + 95 лотов Юаней
```

## Лимиты позиции

### ТАТки (MOEX)

| Лимит | Правило |
|-------|--------|
| Начальный объём | 1–5 лотов (учебный) |
| Рабочий | до 1 000 лотов (безопасный) |
| Предел маржи | Не превышать 60% депозита в позиции |
| Максимум добавок | По мере снижения спреда, малыми частями |

### Предупреждение о маржине

```
Если (используемая маржа / депозит) > MARGIN_WARNING_THRESHOLD:
    → Уведомить оператора
    → Прекратить добавление позиции

Если (используемая маржа / депозит) > MARGIN_CRITICAL_THRESHOLD:
    → НЕМЕДЛЕННО начать закрытие позиции ВРУЧНУЮ
    → Не ждать автоматического margin call брокера
```

**Реальный кейс (урок_15):** маржин-колл = −40% депозита.
Когда брокер закрывает принудительно — очень сильный сквиз.
**Закрывать вручную ДО того, как брокер начнёт принудительное закрытие.**

## Специфика металлов: Золото / Серебро (источник: р9-09, Виктор)

Металлы ведут себя иначе, чем валютные пары — требуют отдельных правил.

### Ключевые отличия

| Параметр | Валюта (юань, EUR/USD) | Металлы (золото/серебро) |
|---------|----------------------|--------------------------|
| Максимальная длительность тренда | Дни | До **33 дней** в одну сторону |
| Средняя волна (серебро) | — | **~15%** |
| Стандартный расчёт лотности | Применим | Нужна отдельная волновая оценка |

### Конструкция 4 ноги (хедж на металлах)

```
Нога 1: Золото  (MOEX / Амойкс)
Нога 2: Серебро (MOEX / Амойкс)
Нога 3: Золото  (Форекс)
Нога 4: Серебро (Форекс)

Соотношение лотности: 100 лотов MOEX : 80 лотов Форекс (= 10 : 8)
```

**Логика:** Золото и серебро высококоррелированы → двойной хедж на двух площадках снижает системный риск при сильном трендовом движении металлов.

### Расчёт лотности для серебра

```python
# Для серебра использовать волновой подход, а не дневные свечи
SILVER_AVG_WAVE_PCT = 0.15   # средняя волна ~15%
GOLD_AVG_WAVE_PCT   = 0.10   # примерная оценка (уточнить по данным)

MOEX_TO_FOREX_RATIO = 0.8    # 100 лотов MOEX = 80 лотов Форекс
```

### Правило дробления для металлов

При расхождении спреда металлов — дробить вход частями:
- Спред стоит на месте, ноги расходятся → по одной ноге прибыль, по второй убыток
- Частичный добор → усредняем обе ноги
- При схождении → прибыль на обеих ногах (или прибыль + небольшой убыток)

## Правила управления позицией

### Что разрешено

| Действие | Условие |
|---------|--------|
| Усреднение (добавка) | ОБЯЗАТЕЛЬНО обе ноги одновременно |
| Частичное закрытие добавок | На волатильности (0.3+ руб движение вверх) |
| Удержание основной позиции | До целевого уровня (5–20 руб) |
| Перенос через ночь | Только если есть финансовый смысл vs стоимость переноса |

### Что запрещено

| Запрет | Почему |
|--------|--------|
| Закрыть одну ногу без второй | Создаёт направленную позицию — это уже не арбитраж (р9-09: «Если вы закрываете одну ногу и пытаетесь тянуть другую — это не арбитраж») |
| Усреднять только убыточную ногу | Нарушает нейтральность, ломает логику коинтеграции |
| Ждать margin call брокера | Принудительное закрытие = гарантированный сквиз |
| Держать unbalanced position | Любая несбалансированность = риск |

### Железное правило (р9-09, Виктор)

> **«Я открываю позицию и закрываю позицию по обеим ногам одновременно. В принципе, на этом всё.»**

В коде MATS это означает: функция `close_position()` всегда закрывает **обе ноги атомарно**. Нет API для закрытия одной ноги без второй.

## Торговля «у единицы» (ratio ≈ 1)

Спред около нуля = зона повышенной волатильности, начало потенциального большого движения.

```
if spread < 0.5:  # "у единицы"
    volatility_multiplier = 1.5  # расширить ожидаемый диапазон
    prefer_long_spread = True    # лонг безопаснее шорта
    extra_margin_reserve = True  # держать дополнительный запас
```

**Наблюдение (урок_15):** «шорт брать опаснее, чем лонг. Тем более мы у единицы — это начало своего. Повышенная волатильность.»

## Стоп-правила

### Когда остановить торговлю

| Ситуация | Действие |
|---------|---------|
| Маржа > CRITICAL% | Закрыть позицию немедленно вручную |
| Спред ушёл против на N рублей | TODO: определить уровень по результатам тестирования |
| API недоступен > X секунд | Остановить новые входы, мониторить позицию |
| Одна нога исполнена, вторая нет > X сек | Экстренное закрытие первой ноги по рынку |
| Суточный убыток > Y% | Остановить торговлю до ручного анализа |

### Параметры конфига

```python
# --- Маржа (из урока 15: маржин-колл = −40% депозита) ---
MARGIN_WARNING_THRESHOLD  = 0.50  # 50% депозита в позиции → алерт
MARGIN_CRITICAL_THRESHOLD = 0.65  # 65% → начинаем закрывать вручную

# --- Суточный стоп ---
MAX_DAILY_LOSS_PCT = 0.04         # 4% суточный стоп (= 1/5 от 20% просадки фропа)

# --- Дисбаланс ног ---
MAX_ONE_LEG_OPEN_SECONDS = 5      # макс. время неисполнения второй ноги

# --- Риск на сделку (р9-09: 1% от депозита при 5 инструментах) ---
DRAWDOWN_LIMIT_PCT      = 0.20    # 20% просадка фропа
N_INSTRUMENTS           = 5       # диверсификация по инструментам
RISK_PER_TRADE_PCT      = DRAWDOWN_LIMIT_PCT / N_INSTRUMENTS  # = 4% (от депо)
# Денежный расчёт: RISK_PER_TRADE_RUB = DEPOSIT_RUB * RISK_PER_TRADE_PCT

# --- Металлы ---
SILVER_AVG_WAVE_PCT     = 0.15    # средняя волна серебра ~15% (р9-09)
MOEX_TO_FOREX_RATIO     = 0.80    # соотношение лотности MOEX:Форекс = 100:80

# --- TODO (заполнить по результатам тестирования) ---
```

## Правила по направлению (шорт vs лонг спреда)

```
Лонг спреда (ЛОНГ TATN + ШОРТ TATNP):
  - Риск снизу ограничен (спред исторически не уходит < 0)
  - Исторический пол: ~0 руб (кроме ковид: −6 руб)
  - РЕКОМЕНДУЕТСЯ для основной позиции

Шорт спреда (ШОРТ TATN + ЛОНГ TATNP):
  - Риск неограничен вверх (спред может уйти на 20+ руб)
  - Использовать только при явном расширении спреда
  - ЗАПРЕЩЁН при спреде < 1 руб (у единицы)
```

## Риск новостных переносов (источник: р9-09, Виктор)

Перед праздниками и длинными выходными — повышенный риск «разрыва» одной ноги.

```
Даже при полном хедже: резкая новость может сдвинуть рынок только по одному инструменту.
→ Одна нога резко уходит, вторая стоит → временный нарушенный баланс.
```

**Правила:**
- Перед праздником/длинными выходными → снизить размер позиции вдвое или закрыть
- Если позиция переносится → обязательно проверить маржу с запасом
