# MATS — Архитектура режима «Полуавтомат»

> Документ описывает структуру и алгоритм системы начиная с v2.0.
> Цель: система находит точку входа/выхода, сигнализирует трейдеру,
> трейдер нажимает «Разрешить» → система выставляет ордера и ведёт позицию.

---

## Компоненты

### 1. Monitor
- Опрашивает T-Invest API каждые 5 сек: `GetLastPrices` для TATN + TATNP
- Параллельно: `GetCandles` для исторического контекста
- Передаёт данные в Signal Engine

### 2. Signal Engine
Вычисляет и оценивает:

| Параметр | Формула |
|----------|---------|
| Спред | `TATN − TATNP` (руб.) |
| Ratio | `TATN / TATNP` |
| Z-score | `(спред − среднее) / стд` за последние N свечей |
| Тренд спреда | Направление последних 5 свечей |

**Условия сигнала ВХОД:**
- Спред ≥ 5.0 руб. (скальп) или ≥ 7.0 руб. (хороший вход)
- Ratio ≥ 1.008
- Z-score > 1.5 (спред выше нормы статистически)
- Тренд спреда: разворот вниз или плоский (начало схождения)

**Условия сигнала ВЫХОД:**
- Спред ≤ 0.5 руб. (схождение)
- Или: достигнут тейк-профит
- Или: достигнут стоп-лосс

### 3. Alert Panel
При сигнале — поверх дашборда всплывает блок:

```
╔══════════════════════════════════════════════╗
║  🚨 СИГНАЛ: LONG SCALP                       ║
╠══════════════════════════════════════════════╣
║  Купить TATNP:   533.80 ₽   × N лотов       ║
║  Продать TATN:   563.80 ₽   × N лотов       ║
║  Спред сейчас:   30.0 ₽                     ║
║  Цель выхода:    0.5 ₽  (схождение)         ║
║  Прогноз прибыли: XXX ₽                     ║
║  Риск (стоп):    −YYY ₽  (−2% депо)        ║
║  R/R:            1 : 3                      ║
╠══════════════════════════════════════════════╣
║  [ ✅ РАЗРЕШИТЬ ]      [ ❌ ПРОПУСТИТЬ ]    ║
╚══════════════════════════════════════════════╝
```

- Панель мигает красным/зелёным
- Можно добавить звуковой сигнал (JavaScript в Streamlit)
- Таймаут: если не нажато за 30 сек → сигнал отменяется

### 4. Order Executor
После нажатия «РАЗРЕШИТЬ»:
1. `asyncio.gather(place_buy, place_sell)` — обе ноги одновременно (правило из р9-09)
2. Тип ордера: лимитный (лучше цена) или рыночный (гарантия исполнения)
3. API: T-Invest `PostOrder`
4. Логирование: время, цены, объём → `logs/trades.csv`

### 5. Position Tracker
После исполнения ордеров:
- Показывает P&L в реальном времени (руб. и %)
- Отслеживает: текущий спред vs спред входа
- Стоп-лосс: автоматически (без подтверждения) при −2% депо
- Тейк-профит: алерт трейдеру → [ЗАКРЫТЬ]
- Время в позиции: счётчик

## Расчёт параметров входа (по риск-менеджменту р9-09)

```
Депо:          10 000 ₽
Макс. просадка: 20% = 2 000 ₽
Частей:        5
1 сделка:      2 000 / 5 = 400 ₽ риска

Стоп-лосс:     вход_спред + 3 руб. (пробой уровня)
Лотность:      400 ₽ / стоп_в_рублях_на_лот
Тейк:          стоп × 3 (R/R = 1:3)
```

## Технические решения

- **Фреймворк:** Streamlit (уже используется)
- **Кнопка «Разрешить»:** `st.button()` + `st.session_state` для хранения состояния
- **Мигание:** CSS-анимация через `st.markdown(unsafe_allow_html=True)`
- **Ордера:** T-Invest REST API `PostOrder` (тот же токен)
- **DRY_RUN:** флаг в `.env` — при `DRY_RUN=true` ордера логируются, но не отправляются
- **Asyncio:** `asyncio.gather` для одновременного выставления обеих ног
