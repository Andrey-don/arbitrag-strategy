# Урок 02: Научил Новичка ТРЕЙДИНГУ за День. Подробное руководство 1

## 1. Краткое резюме

Трейдер с 15-летним опытом на практике обучает новичка основам скальпинга и сбора спреда. Разбирается пошаговый процесс настройки рабочего места: от регистрации на бирже и создания API-ключей до совершения первых прибыльных сделок в торговом приводе. Видео даёт практические навыки работы со стаканом, понимания поведения маркетмейкера и извлечения гарантированной прибыли из рыночной неэффективности на неизвестных токенах.

---

## 2. Ключевые концепции

- **Стакан ордеров** — визуальное отображение лимитных заявок. Снизу (зелёная зона) — заявки на покупку с деньгами, сверху (красная зона) — заявки на продажу с товаром.
- **Лента тиков** — история совершённых сделок: цена и объём в моменте.
- **Спред (Spread)** — разница между лучшей ценой на покупку (зелёной) и лучшей ценой на продажу (красной).
- **Сбор спреда** — стратегия: выставить лимитную заявку на покупку внизу спреда и сразу продать вверху, забирая разницу.
- **Мейкер / Тейкер** — мейкер создаёт рынок (выставляет лимитный ордер), тейкер «бьёт в стакан» (соглашается на существующую цену).
- **Маркетмейкер (алгоритм)** — торговый робот биржи, котирующий стакан на неликвидных активах, покупая и продавая сам у себя для имитации объёма.
- **Биржи третьего эшелона** — азиатские площадки с лояльными условиями (MEXC, Gate, KuCoin, HTX), где торгуются малоизвестные токены без реальных участников.

---

## 3. Практические шаги и алгоритм действий

1. Установить скальперский торговый привод (Tiger Trade, Metascalp, CScalp/Калибри).
2. Зарегистрироваться на бирже третьего эшелона (например, MEXC).
3. Создать API-ключи: разрешить **View account details** + **View order placing**, **запретить трансферы (выводы)**.
4. Вставить API-ключи в настройки торгового привода (отдельно для спота и фьючерсов, указать UID).
5. Пополнить спотовый кошелёк стейблкоинами **USDT в сети TRC20**.
6. Выбрать малоизвестный токен без реальных участников — стакан котирует только робот-маркетмейкер.
7. Выставить лимитную заявку на покупку **внизу спреда** (левая кнопка мыши).
8. После исполнения — сразу выставить лимитную заявку на продажу **вверху спреда** (правая кнопка).
9. Кнопка **Пробел** — мгновенное снятие всех неисполненных заявок.

## 5. Шпаргалка: термин → объяснение → применение в боте

| Термин | Что означает | Как использовать в боте |
|--------|-------------|------------------------|
| Лимитный ордер | Заявка по указанной цене, ждёт исполнения | `create_limit_buy_order()` / `create_limit_sell_order()` |
| Ударить в стакан | Рыночная сделка по лучшей доступной цене | Ордер с перекрытием противоположной стороны стакана |
| Спот | Рынок физического владения активом | Проверять баланс базового актива перед `sell` |
| Ёрш | Цена неликвидного токена гоняется в узком диапазоне | Отслеживать узкий коридор: buy low → sell high |
| Снятие заявок | Аналог кнопки Пробел — отмена всех открытых ордеров | `cancel_all_orders(symbol)` |

## 6. Важные замечания

- **Ограничение объёмов:** нельзя выставлять большие суммы, если маркетмейкер бьёт мелкими частями (по **$2**). Робот не прожуёт большой объём и сдвинет диапазон котирования.
- **Главная ошибка новичков:** пытаться угадать направление рынка. Правильный подход — забирать спред внутри стакана гарантированно, не гадая о трендах.
- **Остатки позиции:** обязательно закрывать позицию на **весь объём** — из-за разницы цен копятся нереализованные остатки токенов.
- **Проблема старта:** **99%** неудач — на этапах установки платформы, заведения API-ключей и покупки крипты.
- **Выбор токенов:** лучше всего — забытые токены, созданные для скама. Там нет реальных трейдеров, работает только алгоритм биржи, который можно обыгрывать.

## 7. Связь с другими уроками

Этот урок закладывает базу механики стакана (сбор спреда в рамках **одной** площадки). Трейдер отмечает: это первый шаг («скальперская часть»). В дальнейшем этот навык потребуется для перехода к настоящему арбитражу — когда стаканов будет несколько и спред будет собираться **между разными биржами**.

## 8. Применение в арбитражном боте

**Биржи для подключения через ccxt:** фокус на азиатских площадках третьего эшелона — `mexc`, `gate`, `kucoin`, `htx`, `bybit`, `bitget`.

**Настройка API-ключей:** разрешить только чтение и выставление ордеров, **запрет трансферов обязателен**.

**Типы ордеров:** исключительно **лимитные мейкерские ордера** для работы со спредом.

```python
import ccxt.async_support as ccxt
import asyncio

async def spread_collector():
    exchange = ccxt.mexc({
        'apiKey': 'YOUR_API_KEY',
        'secret': 'YOUR_SECRET',
        'enableRateLimit': True,
    })

    symbol = 'PLB/USDT'   # неликвидный токен без реальных участников
    amount = 2.0           # мелкий объём — не пугать маркетмейкера

    try:
        ob = await exchange.fetch_order_book(symbol)
        bid = ob['bids'][0][0]  # лучшая цена покупки (низ спреда)
        ask = ob['asks'][0][0]  # лучшая цена продажи (верх спреда)

        # Шаг 1: купить внизу спреда (лимитный мейкер)
        buy_order = await exchange.create_limit_buy_order(symbol, amount, bid)

        # Шаг 2: после исполнения — продать вверху спреда
        sell_order = await exchange.create_limit_sell_order(symbol, amount, ask)

        # Аналог кнопки Пробел — снять все заявки при необходимости
        # await exchange.cancel_all_orders(symbol)

    except Exception as e:
        print(f"Ошибка: {e}")
    finally:
        await exchange.close()

if __name__ == '__main__':
    asyncio.run(spread_collector())
```

> **Важно для бота:** отслеживать остатки позиции после каждой сделки и закрывать точный объём, иначе будут копиться нереализованные остатки токенов.
