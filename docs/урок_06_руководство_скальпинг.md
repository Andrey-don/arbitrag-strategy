# Урок 06: Научил Новичка ТРЕЙДИНГУ за День. Подробное руководство

## 1. Краткое резюме

Трейдер Антон Клевцов (15 лет опыта) обучает новичка Филиппа основам скальпинга и сбору спреда на неликвидных токенах с нуля за один день. К концу сессии Филипп совершил ~100 сделок и заработал **$0.40** на рабочий объём **$2** (~25% к активному торговому капиталу). Ни одной убыточной сделки — потому что он не угадывал тренд, а математически собирал разницу между bid и ask внутри зажатого спреда «глупого» биржевого алгоритма (маркетмейкера).

---

## 2. Концепция стратегии

**Суть заработка:** трейдер не угадывает направление — он зарабатывает на микроскопической разнице между ценой покупки и продажи (спреде) на малоликвидных инструментах. «Наша задача купить за рубль, продать за два».

**Сбор спреда:** постоянное выставление заявок по краям ценового диапазона. Купил по нижней границе — тут же выставил продажу по верхней. «На доллар купил по 70 центов, по 71 центу продал... Таким образом мы будем собирать доли доллара на каждой операции».

**Почему малоизвестные токены:** на биткоине огромная конкуренция. Нужны «всратые» активы, где нет реальных участников. «Нам нужны непопулярные решения, те, которых нет у большинства».

**Маркетмейкер-бот:** биржевой алгоритм, который гоняет цену вверх-вниз для создания видимости активности на мёртвом токене. Трейдер эксплуатирует его прямолинейность: «Нам нужно ему дать возможность нам продать, чтобы мы у него купили. И в него же выйти сверху».

## 3. Технический стек и настройка

### 3.1 Торговый терминал

**Варианты:** Tiger Trade, Metascalp, **CScalp (Калибри)** — в видео используется CScalp.  
**Почему терминал:** мгновенная реакция, заявка ставится кликом мыши — «меньше секунды занимает». Через веб-интерфейс биржи работать невозможно.

### 3.2 Биржи

MEXC, Gate, KuCoin, HTX, Bybit, Bitget — третий эшелон, «они такие очень лояльны ко всему миру». Именно там генерируется огромное количество неизвестных щиткоинов с маркетмейкерами.

### 3.3 Подключение API-ключей (пошагово)

1. Биржа MEXC → Настройки → **Управление API** (API Management)
2. Создать ключи (назвать, например, «Учусь»)
3. Права — поставить галочки:
   - ✅ View account details (смотреть баланс)
   - ✅ View order placing (выставление ордеров)
   - ✅ Торговля
   - ❌ **Трансфер/вывод — категорически запрещено**. «Ставить всё, кроме трансфера. Трансфер тебе нельзя»
4. Для фьючерсов: браузер → F12 → Application → Cookies → скопировать параметр **UID**
5. В CScalp: Настройки → Добавить подключение → вставить API-ключ, Secret-ключ, UID → «Подключить»
6. Настройки → **Экспорт настроек** (сохранить конфиг)

### 3.4 Пополнение счёта

- Сеть: **USDT TRC20**
- Стартовый депозит Филиппа: **$50** (общий баланс **$89**)
- Активный торговый объём: **$2 на сделку**

## 4. Как выбирать тикер

**Признаки правильного тикера:**
- Мёртвый актив, котирует только один робот
- Бьёт мелкими **одинаковыми** объёмами (например, ровно по $2 или $2.7)
- Цена ходит в узком горизонтальном коридоре

**«Ярш» (ёрш):** паттерн на графике, когда алгоритм таскает цену вверх-вниз в узком диапазоне. «Как ёршик, вантуз. Он просто таскает цену вот в этом маленьком... узеньком диапазоне».

**Где искать:**
- MEXC: ~3000 тикеров
- Все биржи суммарно: ~5000–6000 тикеров
- Задача: перебирать, «найти то, что вот так ходит, то, что даёт возможности»

**Размер сделки:** строго минимальный — **$1–2**. «Ты не можешь ставить по 100 долларов, потому что на 100 долларов он уже реагирует. То есть объём сделки в 100 он не прожуёт».

## 5. Механика торговли в стакане

### 5.1 Чтение стакана

| Зона | Цвет | Значение |
|------|------|----------|
| Снизу | Зелёная | Заявки покупателей с деньгами (Bid) |
| Сверху | Красная | Заявки продавцов с товаром (Ask) |
| Шарики | Зелёный | Удар по Ask — рыночная покупка |
| Шарики | Красный | Удар по Bid — рыночная продажа |

**Спред:** разница между лучшей зелёной и лучшей красной ценой. Это и есть наш заработок.

### 5.2 Управление в CScalp

| Действие | Клавиша/кнопка |
|----------|----------------|
| Купить (лимитная заявка) | Левая кнопка мыши в зелёной зоне |
| Продать (лимитная заявка) | Правая кнопка мыши в красной зоне |
| Снять все заявки | **Пробел** |
| Закрыть всю позицию | **Alt** |
| Прокрутить стакан | Колёсико мыши |

### 5.3 Алгоритм действий

1. Найти тикер с «яршом»
2. Купить как можно ниже — **перед крупными объёмами** алгоритма
3. Сразу выставить продажу ближе к красной границе
4. Когда исполнилось — повторить
5. При необходимости закрыть всё: **Alt** → всё выставляется в стакан → **пробел** для отмены, если передумал

### 5.4 Чего НЕ делать

- Не ставить **большой объём** — робот уйдёт из диапазона. «Они уводят диапазон котирования. Он уходит от твоих заявок»
- Не оставлять заявки надолго — «снимать почаще»
- Не угадывать направление. «Ну ты не рулишь процессом»

## 7. Путь масштабирования

**Шаг 1:** Найти тикеры, где движение амплитуднее, маркетмейкер «жирнее» — там ставить уже по **$1000**.

**Шаг 2:** Перейти от пассивного сбора к **активному давлению** на стакан. «Активность — когда ты набираешься, подставляешь его диапазон котирования, сам меняешь, об него закрываешься. Это уже следующий левел».

**Ступени роста:**
1. Скальпинг мелким объёмом (текущий урок)
2. Скальпинг крупным объёмом (жирные тикеры)
3. Межбиржевой арбитраж («с нанесшат битражен» — следующая ступень)

---

## 8. Ошибки большинства трейдеров

- Угадывают направление вместо алгоритмичных действий
- Торгуют эмоционально (FOMO, страх убытка)
- У большинства — **тысяча сделок, тысяча убытков**. Причина: не алгоритм, а азарт
- Боятся технической настройки (API, терминал) → 99% неудач именно здесь

## 9. Глоссарий

| Термин | Значение |
|--------|----------|
| **Спред** | Разница между Bid и Ask — наш заработок |
| **Bid** | Лучшая цена покупателя (зелёная зона, снизу) |
| **Ask** | Лучшая цена продавца (красная зона, сверху) |
| **Стакан** | Визуальная очередь заявок. «Сжатая форма рынка» |
| **Лента тиков** | История проходящих сделок в реальном времени (шарики) |
| **Лимитный ордер** | Отложенная заявка в стакан, ждёт исполнения (мейкер) |
| **Маркет-ордер** | Удар в стакан по текущей цене (тейкер) |
| **Мейкер** | Тот, кто создаёт рынок лимитной заявкой |
| **Тейкер** | Тот, кто бьёт рыночным ордером по чужой лимитке |
| **Ярш (ёрш)** | Паттерн: робот гоняет цену в узком коридоре вверх-вниз |
| **Маркетмейкер** | Биржевой алгоритм, имитирующий торги на «мёртвом» токене |
| **Спот** | Рынок физической поставки: чтобы продать — нужно сначала купить |
| **Фьючерс** | Торговля с плечом, шорт без покупки актива |
| **API-ключ** | Ключ доступа для подключения терминала к бирже |
| **Тикер** | Обозначение торговой пары, например PLB/USDT |
| **Диапазон котирования** | Ценовой коридор, в котором сейчас работает маркетмейкер |
