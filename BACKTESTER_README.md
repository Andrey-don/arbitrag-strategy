## Шаг 0 — Одноразовая настройка

2. Создай **Read-only** токен (торговые операции не нужны)
3. Открой файл `.env` в папке проекта и добавь строку:
   ```

   ```

## Шаг 1 — Загрузка истории (офлайн-режим)

**Двойной клик на `download_history.bat`**

Скрипт скачает **3 месяца** истории TATN и TATNP в двух таймфреймах (5-мин и 1-час)
и сохранит в `data\history\`.

Время загрузки: ~1–3 минуты (зависит от скорости интернета).

После загрузки интернет больше **не нужен** — бэктестер работает офлайн.

### Как добавить новый инструмент (например, Золото)

Открой `src\monitoring\download_history.py`, найди раздел `INSTRUMENTS`:

```python
INSTRUMENTS: dict[str, str] = {
    "TATN":  "BBG004RVFFC0",
    "TATNP": "BBG004S68829",
    # Раскомментируй нужное или добавь своё:
    # "SBER":  "BBG004730N88",
    # "SBERP": "BBG0047315Y7",
    # "GOLD":  "<FIGI>",
}
```

Как найти FIGI любого инструмента:
- Зайди на [tinkoff.ru/invest](https://www.tinkoff.ru/invest/), найди акцию
- Или используй T-Invest API: `FindInstrument` по тикеру

После добавления запусти `download_history.bat` снова — скачаются все инструменты.

## Как пользоваться бэктестером

### Боковая панель (левый сайдбар)

| Параметр | Описание |
|----------|----------|
| **Источник** | `📡 T-Invest API` — загрузка онлайн; `💾 Локальный CSV` — офлайн из файла |
| **Начало / Конец** | Диапазон дат для анализа |
| **Таймфрейм** | 1, 5, 10, 15, 60 мин |
| **Депозит ₽** | Начальный капитал для расчёта метрик |
| **Лотов на сделку** | Фиксированное число лотов (если не используешь авто-расчёт) |
| **Слипадж ₽** | Ухудшение цены входа (моделирует реальное исполнение) |
| **Cooldown, мин** | Пауза после закрытия сделки перед следующей |

**После выбора параметров нажми `📥 Загрузить данные`.**

---

### Вкладка «📊 Бэктест»

Полный прогон стратегии на всей истории за один клик.

**Настройки стратегии:**
- **Спред скальп ≥** — минимальный спред для входа по сигналу LONG_SCALP
- **Ratio скальп ≥** — ratio (TATN/TATNP) для сигнала LONG_SCALP
- **Спред хороший ≥** — спред для сильного сигнала LONG_GOOD
- **Ratio хороший ≥** — ratio для сигнала LONG_GOOD
- **Стоп: доп. к спреду ₽** — стоп-лосс = спред входа + N рублей
- **Тейк: спред схождения ₽** — тейк-профит при схождении спреда до этого уровня

**Нажми `▶ Запустить бэктест`.**

**Результаты:**
- Метрики: Сделок, Winrate, P&L, ROI, Просадка, Sharpe
- Кривая капитала + просадка
- График спреда с точками входа/выхода
- Таблица всех сделок

**Секция «📅 Анализ по сессиям»** (внизу после таблицы):
- Сравнивает утреннюю (10–18 МСК) и вечернюю (19–23 МСК) сессии MOEX
- Ключевой вывод из урока 11б: спред расходится чаще вечером
- Bar-график P&L по часам входа
- Scatter: каждая сделка — точка на графике (час × P&L)

---

### Вкладка «🔍 Оптимизатор»

Grid search по параметрам — находит лучшую комбинацию автоматически.

**Настрой диапазоны:**
- entry_scalp: от/до/шаг (порог входа скальп)
- stop_add: от/до/шаг (размер стопа)
- target: выбери несколько вариантов тейка

**Метрика оптимизации:** Sharpe (рекомендуется), ROI, Winrate или P&L.

**Нажми `🔍 Запустить оптимизацию`.**

**Результаты:**
- Топ-10 лучших комбинаций
- Тепловые карты (entry_scalp × stop_add) для каждого target
- Лучшая найденная комбинация параметров

> Рекомендуется: сначала широкий поиск с шагом 1.0, потом точный с шагом 0.5
> вокруг лучшей найденной точки.

---

### Вкладка «🎮 Тренажёр»

Пошаговое воспроизведение торговли на реальной истории.
**График показывает только прошлое** — будущего не видно.

#### Режимы

**🖐 Полуавтомат** — алгоритм подсвечивает сигналы, ты решаешь:
- `✅ Войти` — открыть позицию
- `⏭ Пропустить` — пропустить сигнал
- `▶ Следующая` — перейти к следующей свече

**🤖 Полный автомат** — алгоритм торгует сам:
- `▶ Следующая` — шаг по одной свече
- `⏩ До конца` — прокрутить всю оставшуюся историю за секунду

#### Параметры (в самой вкладке)
- Депозит, Стоп, Тейк — влияют только на тренажёр, не на бэктест

#### Что показывает тренажёр

В каждой свече:
- Время, Спред, Ratio, Капитал
- Сигнал с цветовым индикатором (🟢/🟡/🔴/⚪)
- Параметры входа: Лотов, Комиссия, Потенциал, Риск, **R/R**
- Если позиция открыта: текущий P&L в реальном времени

График: история до текущей свечи с маркерами входов (▲) и выходов (▼).

Лог и итог: внизу вкладки таблица сделок и финальные метрики.

## Правила входа (стратегия ТАТки)

| Сигнал | Условие | Действие |
|--------|---------|----------|
| LONG_GOOD | Спред ≥ 7 ₽ И Ratio ≥ 1.011 | Сильный вход |
| LONG_SCALP | Спред ≥ 5 ₽ И Ratio ≥ 1.008 | Скальп-вход |
| ALERT | Спред ≥ 2.5 ₽ | Готовиться, наблюдать |
| EXIT | Спред ≤ 0.5 ₽ | Закрывать позицию |
| TAKE_PROFIT | Спред ≥ 20 ₽ | Зона выхода на среднесрочке |

**Фильтр R/R:** вход только если расчётный R/R ≥ 2.0

## Советы по использованию

1. **Начни с Оптимизатора** на 1–2 месяцах истории → найди лучшие параметры
2. **Проверь на другом периоде** (out-of-sample) с найденными параметрами
3. **Сравни сессии** в секции «Анализ по сессиям» — если вечер значительно лучше, торгуй только вечером
4. **Потренируйся в Тренажёре** на полуавтомате — поймёшь, как ты принимаешь решения под давлением
5. **Слипадж 0.2–0.5 ₽** даёт реалистичную картину (рыночные ордера на ТАТках)

## Частые вопросы

**Q: Сколько хранится история в CSV-файлах?**
A: По умолчанию 3 месяца. Изменить в `download_history.py` → `MONTHS_BACK = 3`.

**Q: Можно ли обновить историю (доложить новые данные)?**
A: Пока нет — скрипт перезаписывает файл целиком. Просто запусти `download_history.bat` снова.

**Q: Почему мало данных при 5-мин таймфрейме?**
A: T-Invest API отдаёт максимум 1 день за запрос на мин-таймфреймах. Загрузчик автоматически делает несколько запросов, но это занимает время. Для длинного периода используй 1-час таймфрейм.

**Q: Как добавить пару SBER/SBERP?**
A: В `download_history.py` раскомментируй строки SBER и SBERP → запусти загрузчик. В бэктестере появится новый файл в выпадающем списке CSV. Но бэктестер сейчас заточен под TATN/TATNP — для другой пары потребуется небольшая доработка кода.
