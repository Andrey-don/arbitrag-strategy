**В НАЧАЛЕ каждой сессии — читать ТОЛЬКО SESSION_BRIEF.md и НИЧЕГО БОЛЬШЕ.**
Прочитал SESSION_BRIEF → сказал пользователю статус → спросил "Продолжаем или новая задача?"
Читать другие файлы ТОЛЬКО если пользователь явно запросил конкретную информацию.

**В КОНЦЕ сессии** (команда "обнови SESSION_BRIEF"):
1. Обновить SESSION_BRIEF.md: что сделано, следующий шаг, дата
2. git add → git commit → git push origin main
3. Подтвердить: "SESSION_BRIEF обновлён, изменения запушены"

## Структура проекта

```
arbitrag-strategy/
├── SESSION_BRIEF.md            ← ЕДИНСТВЕННЫЙ файл для начала сессии
├── docs/
│   ├── карта_курса.md          ← Статус всех уроков (обновлять после каждого)

├── src/
│   ├── data/        # Получение данных с бирж (websocket, REST)
│   ├── strategies/  # Логика арбитражных стратегий
│   ├── execution/   # Исполнение ордеров
│   ├── monitoring/  # Мониторинг спредов и возможностей
│   ├── risk/        # Управление рисками
│   └── utils/       # Вспомогательные функции
```

## Правила кода

- `.env` — НЕ коммитить (API ключи)
- Весь async-код через `asyncio` (скорость критична для арбитража)
- Логировать все найденные арбитражные возможности (включая пропущенные)
- Режим `DRY_RUN=true` по умолчанию — реальная торговля только явно
